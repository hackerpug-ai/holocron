DESIGN DECISIONS
-----------------

1. **Visual Hierarchy**
   - Bar chart for score progression (most prominent)
   - Cost analysis below with clear model differentiation
   - Divider separates sections for visual clarity

2. **Color Strategy**
   - Score colors follow traffic light pattern (red → yellow → green)
   - GPT-5 uses primary theme color (darker, premium feel)
   - GPT-5-mini uses secondary theme color (lighter, efficient feel)
   - Theme-aware colors support both light and dark modes

3. **Layout & Spacing**
   - Responsive bar chart adapts to any number of iterations (1-8+)
   - Even spacing between bars using flexbox
   - Consistent padding using theme tokens (spacing.md, spacing.lg)
   - Border radius from theme.radius for visual consistency

4. **Data Structure**
   Extended DeepResearchIteration type with:
   - costCents: Optional cost tracking in cents
   - modelType: 'gpt-5' | 'gpt-5-mini' for model identification
   - startedAt/completedAt: Optional timestamps for duration tracking

5. **Accessibility**
   - testID props for automation testing
   - Semantic color meanings (not just decorative)
   - Clear labels and legends for all visual elements
   - Support for screen readers via Text components

6. **Performance**
   - StyleSheet.create for static styles (optimized)
   - Theme values inlined only where dynamic (color switching)
   - Memoization not needed (component renders only on data change)

TECHNICAL NOTES
---------------

1. **Cost Precision**
   - Costs stored in cents to avoid floating-point issues
   - Formatted to 3 decimal places for display ($0.150)
   - Percentage calculations use integer cent values

2. **Model Type Indicators**
   - Zap icon represents both GPT-5 and GPT-5-mini
   - Distinguished by badge background color (primary vs secondary)
   - Counts show iteration distribution per model

3. **Score Visualization**
   - Bar height proportional to score (0-100% scale)
   - Pending iterations show at 30% opacity
   - Active iterations show at 80% opacity with no score
   - Completed iterations show at 100% opacity

4. **Responsive Design**
   - Bars automatically resize based on iteration count
   - Legend wraps on narrow screens (flexWrap: 'wrap')
   - Progress bars fill available width
   - Works on all device sizes (phone, tablet, desktop)
