# Wire ChatThread to chat-history API with pagination

> Task ID: US-016
> Type: feature:integration
> Priority: P2
> Appetite Fit: medium (fits)
> Assignee: react-native-ui-implementer
> PRD Version: 3.1.0
> Stability: evolving

## CRITICAL CONSTRAINTS

### MUST
- Create a custom hook `hooks/use-chat-history.ts` that fetches messages from the chat-history Edge Function
- Implement cursor-based pagination: initial load + load more on scroll
- Handle loading, error, and empty states appropriately
- Integrate ChatThread component in the chat screen (within Expo Router structure)
- Pass `conversationId` from route params or context to the hook
- Refetch messages when `conversationId` changes
- Use the Supabase client from `lib/supabase.ts` for authenticated requests

### NEVER
- Use useEffect + useState for data fetching (use proper data fetching patterns)
- Hardcode conversation IDs
- Block the UI on pagination loading (show inline loading indicator)
- Lose scroll position when loading more messages

### STRICTLY
- The hook must return: `{ messages, isLoading, isLoadingMore, error, loadMore, hasMore, refetch }`
- Initial page size: 20 messages (matches API default)
- Load more threshold: trigger when 50% from the top (visually) is reached

## SPECIFICATION

**Objective:** Wire the ChatThread component to the chat-history Edge Function, enabling real chat history display with infinite scroll pagination in the main chat screen.

**Success looks like:** Opening a conversation loads its messages from the database and displays them in ChatThread. Scrolling up (toward older messages) triggers pagination loading. The loading indicator appears during load-more. Switching conversations fetches the new conversation's history. Error states display gracefully.

### Code Pattern

```typescript
// hooks/use-chat-history.ts
import { useState, useEffect, useCallback } from 'react'
import { supabase } from '@/lib/supabase'

interface ChatMessage {
  id: string
  conversation_id: string
  role: 'user' | 'agent' | 'system'
  content: string
  message_type: string
  card_data: unknown
  created_at: string
}

interface UseChatHistoryResult {
  messages: ChatMessage[]
  isLoading: boolean
  isLoadingMore: boolean
  error: Error | null
  hasMore: boolean
  loadMore: () => Promise<void>
  refetch: () => Promise<void>
}

export function useChatHistory(conversationId: string | null): UseChatHistoryResult {
  const [messages, setMessages] = useState<ChatMessage[]>([])
  const [isLoading, setIsLoading] = useState(false)
  const [isLoadingMore, setIsLoadingMore] = useState(false)
  const [error, setError] = useState<Error | null>(null)
  const [hasMore, setHasMore] = useState(true)
  const [nextCursor, setNextCursor] = useState<string | null>(null)

  const fetchMessages = useCallback(async (cursor?: string) => {
    if (!conversationId) return

    const isLoadMore = !!cursor
    if (isLoadMore) {
      setIsLoadingMore(true)
    } else {
      setIsLoading(true)
      setMessages([])
      setNextCursor(null)
      setHasMore(true)
    }

    try {
      const params = new URLSearchParams({ conversation_id: conversationId })
      if (cursor) params.append('before', cursor)

      const { data, error: fetchError } = await supabase.functions.invoke('chat-history', {
        method: 'GET',
        // Note: Supabase functions.invoke for GET with query params
        // may need adjustment based on actual SDK usage
      })

      // Alternative: direct fetch
      const response = await fetch(
        `${process.env.EXPO_PUBLIC_SUPABASE_URL}/functions/v1/chat-history?${params}`,
        {
          headers: {
            Authorization: `Bearer ${process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY}`,
          },
        }
      )

      if (!response.ok) {
        throw new Error('Failed to fetch messages')
      }

      const result = await response.json()

      if (isLoadMore) {
        // Append older messages (they come newest-first, so prepend to maintain order)
        setMessages((prev) => [...prev, ...result.messages])
      } else {
        setMessages(result.messages)
      }

      setHasMore(result.has_more)
      setNextCursor(result.next_cursor)
      setError(null)
    } catch (err) {
      setError(err instanceof Error ? err : new Error('Unknown error'))
    } finally {
      setIsLoading(false)
      setIsLoadingMore(false)
    }
  }, [conversationId])

  // Initial fetch when conversationId changes
  useEffect(() => {
    fetchMessages()
  }, [fetchMessages])

  const loadMore = useCallback(async () => {
    if (!hasMore || isLoadingMore || !nextCursor) return
    await fetchMessages(nextCursor)
  }, [fetchMessages, hasMore, isLoadingMore, nextCursor])

  const refetch = useCallback(async () => {
    await fetchMessages()
  }, [fetchMessages])

  return {
    messages,
    isLoading,
    isLoadingMore,
    error,
    hasMore,
    loadMore,
    refetch,
  }
}
```

### Screen Integration

```typescript
// app/(drawer)/(tabs)/index.tsx or screens/ChatScreen.tsx
import { View, StyleSheet, ActivityIndicator } from 'react-native'
import { Text } from 'react-native-paper'
import { useLocalSearchParams } from 'expo-router'
import { ChatThread } from '@/components/chat'
import { useChatHistory } from '@/hooks/use-chat-history'
import { useActiveConversation } from '@/hooks/use-active-conversation' // from Epic 1

export default function ChatScreen() {
  const { conversationId } = useActiveConversation() // or from route params
  const { messages, isLoading, isLoadingMore, error, hasMore, loadMore } = useChatHistory(conversationId)

  if (isLoading) {
    return (
      <View style={styles.centered}>
        <ActivityIndicator size="large" />
      </View>
    )
  }

  if (error) {
    return (
      <View style={styles.centered}>
        <Text variant="bodyLarge">Failed to load messages</Text>
        <Text variant="bodySmall">{error.message}</Text>
      </View>
    )
  }

  return (
    <View style={styles.container}>
      <ChatThread
        messages={messages}
        onLoadMore={loadMore}
        isLoadingMore={isLoadingMore}
        hasMore={hasMore}
      />
      {/* ChatInput will be added in US-017 */}
    </View>
  )
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
  centered: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
  },
})
```

### Anti-pattern (DO NOT)

```typescript
// DO NOT use bare useEffect for data fetching
useEffect(() => {
  fetch(url).then(...)  // WRONG: no loading/error handling pattern
}, [])

// DO NOT hardcode conversation ID
const conversationId = 'abc-123'  // WRONG: must come from route/context

// DO NOT lose messages on pagination
setMessages(newMessages)  // WRONG: must append, not replace
```

## NOTES
- The hook uses a simple useState + useEffect pattern for now. Consider migrating to TanStack Query or SWR for better caching in a future iteration.
- The Supabase JS client's `functions.invoke` may not support GET with query params cleanly; direct fetch is shown as an alternative.
- Cursor-based pagination ensures consistent results even as new messages are added.
- The `refetch` function is useful after sending a new message to immediately show it (until real-time is added in US-018).
- Error boundary at the screen level would be a good future enhancement.
