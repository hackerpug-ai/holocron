# Build chat-send Edge Function with basic agent response routing

> Task ID: US-011
> Type: feature
> Priority: P1
> Appetite Fit: medium (fits)
> Assignee: ai-tooling-implementer
> PRD Version: 3.1.0
> Stability: semi-stable

## CRITICAL CONSTRAINTS

### MUST
- Implement a single Edge Function at `supabase/functions/chat-send/index.ts`
- Accept POST requests with JSON body: `{ conversation_id: string, content: string, message_type: 'text' | 'slash_command' }`
- Validate `conversation_id` exists in the `conversations` table before processing
- Persist the user's message to `chat_messages` with role 'user' immediately
- Update the parent conversation's `updated_at` and `last_message_preview` after user message insert
- Generate a basic agent response for `message_type: 'text'` (echo-style or simple acknowledgment for MVP)
- Persist the agent's response to `chat_messages` with role 'agent' and type 'text'
- Return both the user message ID and agent response in the response body
- Return HTTP 200 for success, 400 for validation errors, 404 for conversation not found, 500 for server errors
- Set CORS headers on all responses

### NEVER
- Parse slash commands in this task (slash command routing is Epic 3, US-019)
- Call external AI APIs (Claude, OpenAI) for agent responses in this task; use a simple stub response
- Block on long-running operations; the response must be immediate
- Store the service role key in code; read from `Deno.env.get()`

### STRICTLY
- The agent response must be generated synchronously and returned in the same request/response cycle
- The `last_message_preview` must be truncated to 100 characters max
- Response shape must be: `{ user_message_id: string, agent_messages: AgentMessage[] }`
- Use TypeScript with strict typing; reuse types from `lib/types/chat.ts` concepts (define inline for Deno)

## SPECIFICATION

**Objective:** Build the `chat-send` Edge Function that receives a user message, persists it, generates a basic agent response, persists the response, and returns both to the client — enabling the core send-message flow for Epic 2.

**Success looks like:** POST to `/functions/v1/chat-send` with a valid conversation_id and content returns a JSON response containing the user's message ID and an array of agent messages. The user message appears in `chat_messages` with role 'user', the agent response appears with role 'agent', and the conversation's `updated_at` and `last_message_preview` are updated. Edge cases (empty content, missing conversation, invalid body) return appropriate error responses.

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|

| 4 | A conversation does NOT exist | POST with non-existent `conversation_id` | Response is 404 with `{ error: "Conversation not found" }` | `curl ... -d '{"conversation_id":"00000000-0000-0000-0000-000000000000",...}'` |

### Code Pattern

```typescript
// supabase/functions/chat-send/index.ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'
import { corsHeaders } from '../_shared/cors.ts'

interface ChatSendRequest {
  conversation_id: string
  content: string
  message_type: 'text' | 'slash_command'
}

interface AgentMessage {
  id: string
  role: 'agent'
  content: string
  message_type: 'text' | 'result_card' | 'progress' | 'error'
  card_data?: unknown
}

Deno.serve(async (req: Request) => {
  if (req.method === 'OPTIONS') {
    return new Response(null, { status: 204, headers: corsHeaders })
  }

  if (req.method !== 'POST') {
    return jsonResponse({ error: 'Method not allowed' }, 405)
  }

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!,
  )

  try {
    const body: ChatSendRequest = await req.json()

    // Validate required fields
    if (!body.conversation_id) {
      return jsonResponse({ error: 'conversation_id is required' }, 400)
    }
    if (!body.content || body.content.trim() === '') {
      return jsonResponse({ error: 'Message content cannot be empty' }, 400)
    }

    // Check conversation exists
    const { data: conversation, error: convError } = await supabase
      .from('conversations')
      .select('id')
      .eq('id', body.conversation_id)
      .single()

    if (convError || !conversation) {
      return jsonResponse({ error: 'Conversation not found' }, 404)
    }

    // Insert user message
    const { data: userMessage, error: userError } = await supabase
      .from('chat_messages')
      .insert({
        conversation_id: body.conversation_id,
        role: 'user',
        content: body.content,
        message_type: body.message_type || 'text',
      })
      .select()
      .single()

    if (userError) throw userError

    // Update conversation metadata
    const preview = body.content.length > 100
      ? body.content.slice(0, 97) + '...'
      : body.content

    await supabase
      .from('conversations')
      .update({
        updated_at: new Date().toISOString(),
        last_message_preview: preview,
      })
      .eq('id', body.conversation_id)

    // Generate basic agent response (stub for MVP)
    const agentContent = generateAgentResponse(body.content)

    const { data: agentMessage, error: agentError } = await supabase
      .from('chat_messages')
      .insert({
        conversation_id: body.conversation_id,
        role: 'agent',
        content: agentContent,
        message_type: 'text',
      })
      .select()
      .single()

    if (agentError) throw agentError

    return jsonResponse({
      user_message_id: userMessage.id,
      agent_messages: [{
        id: agentMessage.id,
        role: 'agent',
        content: agentMessage.content,
        message_type: agentMessage.message_type,
      }],
    })

  } catch (error) {
    console.error('chat-send error:', error)
    return jsonResponse({ error: 'Internal server error' }, 500)
  }
})

function generateAgentResponse(userContent: string): string {
  // Basic stub response for MVP - will be replaced with real AI in future epics
  if (userContent.toLowerCase().includes('hello')) {
    return "Hello! I'm your research assistant. How can I help you today?"
  }
  if (userContent.toLowerCase().includes('help')) {
    return "I can help you search your knowledge base, conduct research, and manage articles. Try asking me a question!"
  }
  return `I received your message. Full AI responses will be available soon!`
}

function jsonResponse(body: unknown, status = 200): Response {
  return new Response(JSON.stringify(body), {
    status,
    headers: { ...corsHeaders, 'Content-Type': 'application/json' },
  })
}
```

## NOTES
- The `generateAgentResponse` function is a simple stub. Epic 3+ will add slash command parsing and real AI integration.
- For `message_type: 'slash_command'`, this task still processes it as a text message. The actual slash command routing is US-019.
- The conversation's `updated_at` is updated to enable proper "most recent" sorting in the drawer.
- `last_message_preview` is the user's message (not agent's) for drawer display.
- Consider using a transaction for the user message insert + conversation update, but Supabase client doesn't support transactions directly; if atomicity is critical, use an RPC function in a future iteration.
