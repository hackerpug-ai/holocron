# Build chat-history Edge Function with cursor-based pagination

> Task ID: US-012
> Type: feature
> Priority: P1
> Appetite Fit: medium (fits)
> Assignee: supabase-implementer
> PRD Version: 3.1.0
> Stability: stable

## CRITICAL CONSTRAINTS

### MUST
- Accept GET requests with conversation_id, limit, before query params
- Validate conversation_id is valid UUID format
- Verify conversation exists before fetching messages
- Return messages ordered by created_at DESC (newest first)
- Implement cursor-based pagination using created_at timestamp
- Return has_more boolean and next_cursor string

### NEVER
- Accept POST/PATCH/DELETE requests (GET only)
- Use offset-based pagination (causes inconsistency)
- Return messages without conversation validation
- Return unlimited results (enforce max limit)

### STRICTLY
- Clamp limit between 1 and 100 (default: 20)
- Use 'before' parameter as cursor (ISO timestamp)
- Fetch limit+1 to determine has_more efficiently

## SPECIFICATION

**Objective:** Create the chat-history Edge Function that returns paginated chat messages for infinite scroll, using cursor-based pagination for efficient loading of older messages.

**Success looks like:** GET /functions/v1/chat-history?conversation_id=X&limit=20&before=timestamp returns messages array, has_more boolean, and next_cursor for pagination.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | A conversation with 50 messages | GET is sent with limit=20 | Response is 200 with exactly 20 messages, has_more=true, and next_cursor set | `curl -X GET '$SUPABASE_URL/functions/v1/chat-history?conversation_id=...&limit=20'` |
| 2 | A conversation with 50 messages and first page loaded | GET is sent with before=next_cursor | Response contains the next 20 older messages with no duplicates | `curl -X GET '$SUPABASE_URL/functions/v1/chat-history?conversation_id=...&limit=20&before=$CURSOR'` |

| 4 | An invalid conversation_id UUID format | GET is sent with invalid UUID | Response is 400 with error 'Invalid conversation_id format' | `curl -X GET '$SUPABASE_URL/functions/v1/chat-history?conversation_id=not-a-uuid'` |

### Code Pattern

```typescript
// supabase/functions/chat-history/index.ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'
import { corsHeaders } from '../_shared/cors.ts'

interface ChatHistoryResponse {
  messages: Array<{
    id: string
    role: string
    content: string
    message_type: string
    card_data: unknown | null
    created_at: string
  }>
  has_more: boolean
  next_cursor: string | null
}

const UUID_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i

Deno.serve(async (req: Request) => {
  if (req.method === 'OPTIONS') {
    return new Response(null, { status: 204, headers: corsHeaders })
  }

  if (req.method !== 'GET') {
    return jsonResponse({ error: 'Method not allowed' }, 405)
  }

  const url = new URL(req.url)
  const conversation_id = url.searchParams.get('conversation_id')
  const limitParam = url.searchParams.get('limit')
  const before = url.searchParams.get('before')

  // Validate conversation_id
  if (!conversation_id) {
    return jsonResponse({ error: 'conversation_id is required' }, 400)
  }
  if (!UUID_REGEX.test(conversation_id)) {
    return jsonResponse({ error: 'Invalid conversation_id format' }, 400)
  }

  // Parse and clamp limit
  let limit = parseInt(limitParam || '20', 10)
  if (isNaN(limit) || limit < 1) limit = 20
  if (limit > 100) limit = 100

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  try {
    // Verify conversation exists
    const { data: conv, error: convError } = await supabase
      .from('conversations')
      .select('id')
      .eq('id', conversation_id)
      .single()

    if (convError || !conv) {
      return jsonResponse({ error: 'Conversation not found' }, 404)
    }

    // Build query - fetch limit+1 to determine has_more
    let query = supabase
      .from('chat_messages')
      .select('id, role, content, message_type, card_data, created_at')
      .eq('conversation_id', conversation_id)
      .order('created_at', { ascending: false })
      .limit(limit + 1)

    if (before) {
      query = query.lt('created_at', before)
    }

    const { data: messages, error: msgError } = await query

    if (msgError) {
      console.error('chat-history query error:', msgError)
      return jsonResponse({ error: 'Failed to fetch messages' }, 500)
    }

    // Determine has_more and slice to limit
    const has_more = messages.length > limit
    const result = has_more ? messages.slice(0, limit) : messages
    const next_cursor = has_more ? result[result.length - 1].created_at : null

    return jsonResponse({
      messages: result,
      has_more,
      next_cursor
    })
  } catch (error) {
    console.error('chat-history error:', error)
    return jsonResponse({ error: 'Internal server error' }, 500)
  }
})

function jsonResponse(body: unknown, status = 200): Response {
  return new Response(JSON.stringify(body), {
    status,
    headers: { ...corsHeaders, 'Content-Type': 'application/json' }
  })
}
```

### Anti-pattern (DO NOT)

```typescript
// DO NOT use offset pagination
.range(offset, offset + limit)  // WRONG: causes inconsistency with new messages

// DO NOT allow unlimited results
const limit = parseInt(limitParam)  // WRONG: could be NaN or huge number

// DO NOT skip conversation validation
// Missing validation lets users probe for any conversation_id
```

## CODING STANDARDS

- Parse query params from URL object
- Validate UUID format with regex
- Use .lt() for cursor comparison
- Fetch limit+1 rows, slice to determine has_more
- Return ISO timestamp string as next_cursor

## NOTES

- Cursor pagination uses created_at timestamp for stability across page loads
- Index on (conversation_id, created_at DESC) ensures efficient queries
- The before cursor is exclusive (lt, not lte) to avoid duplicates
- Messages are returned newest-first; client reverses for display
