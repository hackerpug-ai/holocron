## CRITICAL CONSTRAINTS

### MUST
- Create three visual variants: user, agent, system
- Use NativeWind utility classes for all styling (no inline style objects for static layout)
- Use `cn()` from `@/lib/utils` for conditional class merging
- Import `Text` from `@/components/ui/text` -- never from `react-native`
- Include `testID` on every interactive and structural element
- Co-locate a Storybook story file (`MessageBubble.stories.tsx`) as a sibling
- Use mock data for all stories -- no API calls, no hooks that fetch data
- Support variable content length (short messages and multi-paragraph)

### NEVER
- Never hardcode hex colors -- use NativeWind semantic tokens (`bg-background`, `text-foreground`, `bg-muted`, etc.)
- Never hardcode spacing values -- use Tailwind spacing classes (`p-3`, `gap-2`, `mt-1`, etc.)
- Never import `Text` from `react-native` -- always use `@/components/ui/text`
- Never add API fetching logic -- this is a design-only task with mock data
- Never use `useCallback` or `useMemo` unless meeting the specific criteria in CLAUDE.md

### STRICTLY
- All props must be TypeScript interfaces with JSDoc comments
- Component file must be named `MessageBubble.tsx` and export a named function `MessageBubble`
- Story file must follow the `Meta<typeof MessageBubble>` pattern from existing stories
- User messages align right with primary background
- Agent messages align left with surface/muted background
- System messages are centered with subtle styling

## SPECIFICATION

**Objective:** Create a `MessageBubble` component that renders chat messages with distinct visual styles for user, agent, and system message roles, following common chat UI patterns.

**Success looks like:** A fully rendered message bubble in Storybook showing all three variants with proper alignment, colors, and typography. All story variants render without errors and the play function passes.

### Code Pattern
```typescript
// components/chat/MessageBubble.tsx
import { View } from 'react-native'
import { Text } from '@/components/ui/text'
import { cn } from '@/lib/utils'

export type MessageRole = 'user' | 'agent' | 'system'

export interface MessageBubbleProps {
  /** The message content to display */
  content: string
  /** The role determines visual styling and alignment */
  role: MessageRole
  /** Optional timestamp to display below the message */
  timestamp?: string
  /** Optional testID prefix for testing */
  testID?: string
}

export function MessageBubble({ content, role, timestamp, testID = 'message-bubble' }: MessageBubbleProps) {
  const isUser = role === 'user'
  const isSystem = role === 'system'

  return (
    <View
      testID={testID}
      className={cn(
        'flex-row',
        isUser && 'justify-end',
        isSystem && 'justify-center',
        !isUser && !isSystem && 'justify-start'
      )}
    >
      <View
        testID={`${testID}-container`}
        className={cn(
          'rounded-2xl px-4 py-2 max-w-[80%]',
          isUser && 'bg-primary',
          isSystem && 'bg-transparent',
          !isUser && !isSystem && 'bg-muted'
        )}
      >
        <Text
          testID={`${testID}-content`}
          className={cn(
            isUser && 'text-primary-foreground',
            isSystem && 'text-muted-foreground text-sm text-center',
            !isUser && !isSystem && 'text-foreground'
          )}
        >
          {content}
        </Text>
        {timestamp && (
          <Text
            testID={`${testID}-timestamp`}
            className={cn(
              'text-xs mt-1',
              isUser ? 'text-primary-foreground/70' : 'text-muted-foreground'
            )}
          >
            {timestamp}
          </Text>
        )}
      </View>
    </View>
  )
}
```
