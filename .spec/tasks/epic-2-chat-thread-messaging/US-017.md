# Wire ChatInput to chat-send API with optimistic updates

> Task ID: US-017
> Type: feature:integration
> Priority: P2
> Appetite Fit: medium (fits)
> Assignee: react-native-ui-implementer
> PRD Version: 3.1.0
> Stability: evolving

## CRITICAL CONSTRAINTS

### MUST
- Create a custom hook `hooks/use-send-message.ts` that sends messages via the chat-send Edge Function
- Implement optimistic update: show user message immediately before API response
- Handle the agent's response from the API and add it to the message list
- Clear the input field after successful send
- Show error state if send fails (with option to retry)
- Integrate ChatInput component in the chat screen alongside ChatThread
- Disable the input during sending to prevent double-sends

### NEVER
- Block the UI waiting for API response (optimistic update is required)
- Allow sending while a previous send is in progress
- Lose the user's message if send fails (keep it for retry)

### STRICTLY
- Optimistic message must have a temporary ID (e.g., `temp-{timestamp}`)
- On success: replace temp message with real message, add agent response(s)
- On failure: mark optimistic message as failed, show retry option
- Keyboard should dismiss after send

## SPECIFICATION

**Objective:** Wire the ChatInput component to the chat-send Edge Function with optimistic updates, enabling users to send messages and see immediate feedback while the agent processes.

**Success looks like:** User types a message and taps send. The message appears immediately in the chat thread (optimistic). The API processes and returns the agent's response. The agent's response appears below the user's message. If the send fails, an error state is shown with retry capability. The input clears after successful send.

### Code Pattern

```typescript
// hooks/use-send-message.ts
import { useState, useCallback } from 'react'
import { supabase } from '@/lib/supabase'

interface ChatMessage {
  id: string
  conversation_id: string
  role: 'user' | 'agent' | 'system'
  content: string
  message_type: string
  created_at: string
  _status?: 'pending' | 'sent' | 'failed'  // Client-only status
}

interface UseSendMessageResult {
  sendMessage: (content: string) => Promise<void>
  isSending: boolean
  error: Error | null
}

interface UseSendMessageOptions {
  conversationId: string
  onOptimisticMessage: (message: ChatMessage) => void
  onMessageSent: (userMessage: ChatMessage, agentMessages: ChatMessage[]) => void
  onError: (tempId: string, error: Error) => void
}

export function useSendMessage({
  conversationId,
  onOptimisticMessage,
  onMessageSent,
  onError,
}: UseSendMessageOptions): UseSendMessageResult {
  const [isSending, setIsSending] = useState(false)
  const [error, setError] = useState<Error | null>(null)

  const sendMessage = useCallback(async (content: string) => {
    if (!content.trim() || !conversationId || isSending) return

    const tempId = `temp-${Date.now()}`
    const tempMessage: ChatMessage = {
      id: tempId,
      conversation_id: conversationId,
      role: 'user',
      content: content.trim(),
      message_type: 'text',
      created_at: new Date().toISOString(),
      _status: 'pending',
    }

    // Optimistic update
    onOptimisticMessage(tempMessage)
    setIsSending(true)
    setError(null)

    try {
      const response = await fetch(
        `${process.env.EXPO_PUBLIC_SUPABASE_URL}/functions/v1/chat-send`,
        {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
            Authorization: `Bearer ${process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY}`,
          },
          body: JSON.stringify({
            conversation_id: conversationId,
            content: content.trim(),
            message_type: 'text',
          }),
        }
      )

      if (!response.ok) {
        const errorData = await response.json()
        throw new Error(errorData.error || 'Failed to send message')
      }

      const result = await response.json()

      // Build confirmed user message
      const confirmedUserMessage: ChatMessage = {
        ...tempMessage,
        id: result.user_message_id,
        _status: 'sent',
      }

      // Build agent messages
      const agentMessages: ChatMessage[] = result.agent_messages.map((msg: any) => ({
        id: msg.id,
        conversation_id: conversationId,
        role: msg.role,
        content: msg.content,
        message_type: msg.message_type,
        created_at: new Date().toISOString(),
      }))

      onMessageSent(confirmedUserMessage, agentMessages)
    } catch (err) {
      const sendError = err instanceof Error ? err : new Error('Send failed')
      setError(sendError)
      onError(tempId, sendError)
    } finally {
      setIsSending(false)
    }
  }, [conversationId, isSending, onOptimisticMessage, onMessageSent, onError])

  return { sendMessage, isSending, error }
}
```

### Screen Integration

```typescript
// app/(drawer)/(tabs)/index.tsx (partial - ChatInput section)
import { useState, useCallback } from 'react'
import { View, StyleSheet, Keyboard } from 'react-native'
import { ChatThread, ChatInput } from '@/components/chat'
import { useChatHistory } from '@/hooks/use-chat-history'
import { useSendMessage } from '@/hooks/use-send-message'

export default function ChatScreen() {
  const { conversationId } = useActiveConversation()
  const [inputValue, setInputValue] = useState('')

  const {
    messages,
    setMessages,  // Need to expose this or use reducer
    isLoading,
    isLoadingMore,
    hasMore,
    loadMore,
  } = useChatHistory(conversationId)

  const handleOptimisticMessage = useCallback((message) => {
    setMessages((prev) => [message, ...prev])  // Prepend (newest first)
  }, [])

  const handleMessageSent = useCallback((userMessage, agentMessages) => {
    setMessages((prev) => {
      // Replace temp message with confirmed, add agent messages
      const withoutTemp = prev.filter((m) => !m.id.startsWith('temp-'))
      return [
        ...agentMessages.reverse(),  // Agent messages newest first
        userMessage,
        ...withoutTemp.slice(1),  // Rest of messages
      ]
    })
  }, [])

  const handleError = useCallback((tempId, error) => {
    setMessages((prev) =>
      prev.map((m) =>
        m.id === tempId ? { ...m, _status: 'failed' } : m
      )
    )
  }, [])

  const { sendMessage, isSending } = useSendMessage({
    conversationId,
    onOptimisticMessage: handleOptimisticMessage,
    onMessageSent: handleMessageSent,
    onError: handleError,
  })

  const handleSend = useCallback(() => {
    if (inputValue.trim()) {
      sendMessage(inputValue)
      setInputValue('')
      Keyboard.dismiss()
    }
  }, [inputValue, sendMessage])

  return (
    <View style={styles.container}>
      <ChatThread
        messages={messages}
        onLoadMore={loadMore}
        isLoadingMore={isLoadingMore}
        hasMore={hasMore}
      />
      <ChatInput
        value={inputValue}
        onChangeText={setInputValue}
        onSend={handleSend}
        disabled={isSending}
      />
    </View>
  )
}
```

## NOTES
- The optimistic update pattern shows the user message immediately for perceived speed.
- The `_status` field is client-only metadata for tracking pending/sent/failed states.
- Consider using a reducer instead of multiple callbacks for cleaner state management in a future refactor.
- Real-time subscriptions (Supabase Realtime) can be added in US-018 to receive messages pushed by the server instead of relying solely on the API response.
