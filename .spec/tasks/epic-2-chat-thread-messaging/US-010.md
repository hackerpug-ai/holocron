## CRITICAL CONSTRAINTS

### MUST
- Create the `message_role` ENUM type with values: 'user', 'agent', 'system'
- Create the `message_type` ENUM type with values: 'text', 'slash_command', 'result_card', 'progress', 'error'
- Create the `chat_messages` table matching the exact schema in PRD §11 (Technical Requirements)
- Add foreign key constraint on `conversation_id` referencing `conversations(id)` with `ON DELETE CASCADE`
- Create index `idx_chat_messages_conversation` on `(conversation_id, created_at DESC)` for paginated history queries
- Add optional foreign keys to `research_sessions(id)` and `documents(id)` for future epic linking (nullable, no cascade)
- Generate TypeScript types in `lib/types/chat.ts` matching the database schema
- Migration file must be named with timestamp prefix: `{timestamp}_create_chat_messages.sql`

### NEVER
- Modify the existing `conversations` table from US-002; only reference it via FK
- Create RLS policies; this is a single-user personal app using service role key
- Use UUIDs for document_id FK (documents table uses SERIAL PRIMARY KEY integer)
- Add triggers or computed columns; keep the schema simple

### STRICTLY
- The `card_data` column must be `JSONB` (not JSON) for indexing capability
- The `content` column must be `TEXT NOT NULL` (message body is required)
- The `created_at` column must default to `NOW()` and be `TIMESTAMPTZ`
- TypeScript types must use branded types for UUIDs where appropriate

## SPECIFICATION

### Code Pattern

**Migration SQL:**
```sql
-- supabase/migrations/{timestamp}_create_chat_messages.sql

-- Create ENUM types
CREATE TYPE message_role AS ENUM ('user', 'agent', 'system');
CREATE TYPE message_type AS ENUM ('text', 'slash_command', 'result_card', 'progress', 'error');

-- Create chat_messages table
CREATE TABLE chat_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  conversation_id UUID NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
  role message_role NOT NULL,
  content TEXT NOT NULL,
  message_type message_type NOT NULL,
  card_data JSONB,
  session_id UUID,  -- FK to research_sessions added when that table exists
  document_id INTEGER,  -- FK to documents added when linking is needed
  created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

-- Index for paginated history queries (newest first within conversation)
CREATE INDEX idx_chat_messages_conversation ON chat_messages(conversation_id, created_at DESC);
```

**TypeScript Types:**
```typescript
// lib/types/chat.ts

export type MessageRole = 'user' | 'agent' | 'system'
export type MessageType = 'text' | 'slash_command' | 'result_card' | 'progress' | 'error'

export interface CardData {
  card_type: 'article' | 'research_result' | 'stats' | 'session' | 'category_list'
  title: string
  snippet?: string
  category?: string
  confidence_score?: number
  source_count?: number
  document_id?: number
  session_id?: string
  metadata?: Record<string, unknown>
}

export interface ChatMessage {
  id: string
  conversation_id: string
  role: MessageRole
  content: string
  message_type: MessageType
  card_data: CardData | null
  session_id: string | null
  document_id: number | null
  created_at: string
}

export interface ChatMessageInsert {
  conversation_id: string
  role: MessageRole
  content: string
  message_type: MessageType
  card_data?: CardData | null
  session_id?: string | null
  document_id?: number | null
}
```

### Anti-pattern (DO NOT)

```sql
-- DO NOT use JSON instead of JSONB
card_data JSON,  -- WRONG: use JSONB for indexing

-- DO NOT create without CASCADE
conversation_id UUID NOT NULL REFERENCES conversations(id),  -- WRONG: missing ON DELETE CASCADE

-- DO NOT use wrong index direction
CREATE INDEX idx_chat_messages_conversation ON chat_messages(conversation_id, created_at ASC);  -- WRONG: should be DESC for recent-first queries
```
