- Never modify the MessageBubble component

### STRICTLY
- All props must be TypeScript interfaces with JSDoc comments
- Component file must be named `ChatThread.tsx` and export a named function `ChatThread`
- Messages are passed in chronological order (oldest first) and displayed with newest at bottom
- FlatList must be inverted to show newest messages at the bottom
- Loading indicator must appear at top when fetching older messages

## SPECIFICATION

**Objective:** Create a `ChatThread` component that renders a scrollable list of chat messages using `MessageBubble` components, with auto-scroll to newest messages and infinite scroll support for loading history.

**Success looks like:** A fully rendered chat thread in Storybook showing multiple messages with proper spacing, auto-scroll behavior on new messages, and a loading indicator during pagination. All story variants render without errors.

### Code Pattern
```typescript
// components/chat/ChatThread.tsx
import { FlatList, View, ActivityIndicator } from 'react-native'
import { Text } from '@/components/ui/text'
import { MessageBubble, MessageRole } from '@/components/chat/MessageBubble'
import { cn } from '@/lib/utils'
import { useRef, useEffect } from 'react'

export interface ChatMessage {
  id: string
  content: string
  role: MessageRole
  created_at: string
}

export interface ChatThreadProps {
  /** Array of messages to display (oldest first) */
  messages: ChatMessage[]
  /** Whether older messages are being loaded */
  isLoading?: boolean
  /** Callback when user scrolls to top (load older messages) */
  onLoadMore?: () => void
  /** Whether there are more messages to load */
  hasMore?: boolean
}

export function ChatThread({ messages, isLoading, onLoadMore, hasMore }: ChatThreadProps) {
  const flatListRef = useRef<FlatList>(null)

  // Auto-scroll to bottom when new messages arrive
  useEffect(() => {
    if (messages.length > 0) {
      flatListRef.current?.scrollToEnd({ animated: true })
    }
  }, [messages.length])

  const handleEndReached = () => {
    if (hasMore && !isLoading && onLoadMore) {
      onLoadMore()
    }
  }

  const renderMessage = ({ item }: { item: ChatMessage }) => (
    <View className="px-4 py-1" testID={`message-${item.id}`}>
      <MessageBubble
        content={item.content}
        role={item.role}
        timestamp={formatTimestamp(item.created_at)}
        testID={`bubble-${item.id}`}
      />
    </View>
  )

  if (messages.length === 0) {
    return (
      <View className="flex-1 items-center justify-center" testID="chat-thread-empty">
        <Text className="text-muted-foreground">No messages yet</Text>
      </View>
    )
  }

  return (
    <FlatList
      ref={flatListRef}
      testID="chat-thread"
      data={messages}
      renderItem={renderMessage}
      keyExtractor={(item) => item.id}
      inverted={false}
      onEndReached={handleEndReached}
      onEndReachedThreshold={0.1}
      ListHeaderComponent={
        isLoading ? (
          <View className="py-4 items-center" testID="chat-thread-loading">
            <ActivityIndicator />
          </View>
        ) : null
      }
      contentContainerStyle={{ paddingVertical: 8 }}
    />
  )
}

function formatTimestamp(isoString: string): string {
  // Simple formatting - US-018 will enhance this
  const date = new Date(isoString)
  return date.toLocaleTimeString([], { hour: '2-digit', minute: '2-digit' })
}
```

## NOTES
- FlatList `inverted` is NOT used here because messages are in chronological order and we scroll to end
- The `onLoadMore` fires at the TOP of the list (where older messages load) via `onEndReached` with inverted=false but reverse scroll
- Auto-scroll to bottom should only happen for genuinely new messages, not when loading history
- US-016 will wire this to the real API with cursor-based pagination
