## CRITICAL CONSTRAINTS

### MUST
- Create a `useConversations` hook that encapsulates all Supabase conversation CRUD operations
- Use `supabaseAdmin` client from `lib/supabase.ts` for all database operations (service role key, dev mode)
- Handle loading, error, and empty states explicitly in the hook return value
- Wire `DrawerContent` to real conversation data from the `useConversations` hook
- Wire "New Chat" compose button to create a new conversation and navigate to it
- Wire conversation row tap to switch the active conversation and close the drawer
- Use Expo Router navigation (`router.navigate`, `router.replace`) for conversation switching
- Close the drawer after creating a new conversation or switching conversations

### NEVER
- Never call Supabase directly from components -- all data access goes through the `useConversations` hook
- Never store conversation data in component-local state that duplicates the hook state (no state syncing anti-pattern)
- Never skip loading/error states -- every query must show appropriate feedback
- Never use `useCallback` or `useMemo` without meeting the criteria in CLAUDE.md
- Never modify the existing UI primitives in `components/ui/`
- Never modify `DrawerHeader.tsx` or `ConversationRow.tsx` -- only modify `DrawerContent.tsx`

### STRICTLY
- The hook must return `{ conversations, activeConversationId, isLoading, error, createConversation, switchConversation, refetch }`
- Conversations must be sorted by most recent activity (last message timestamp, then created_at)
- New conversations get the default title "New Chat"
- The `conversations` Supabase table schema must be documented in the hook file comments
- All Supabase queries must include error handling with meaningful error messages

## SPECIFICATION

**Objective:** Wire the `DrawerContent` component to real Supabase conversation data by creating a `useConversations` hook and integrating it into the drawer layout, enabling users to list, create, and switch between conversations.

**Success looks like:** Opening the drawer shows real conversations fetched from Supabase. Tapping the compose button creates a new conversation in the database, adds it to the list, navigates to it, and closes the drawer. Tapping a conversation row switches to it, closes the drawer, and the chat screen loads that conversation's context.

### Supabase Table Schema (Expected)
```sql
-- conversations table
CREATE TABLE conversations (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title TEXT NOT NULL DEFAULT 'New Chat',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- messages table (referenced for last_message preview)
CREATE TABLE messages (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  conversation_id UUID REFERENCES conversations(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system')),
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### Interaction Notes
- On drawer open: fetch conversations with last message preview (join or subquery)
- On compose tap: `INSERT INTO conversations (title) VALUES ('New Chat')` -> navigate to new ID -> close drawer
- On row tap: set active conversation ID -> navigate to conversation route -> close drawer
- Drawer close: use `navigation.closeDrawer()` from `@react-navigation/drawer` or Expo Router equivalent
- Loading state: show skeleton/spinner in the conversation list area (not full screen)
- Error state: show error message with "Tap to retry" in the conversation list area

```typescript
// In app/(drawer)/_layout.tsx -- wire the hook to DrawerContent
import { useConversations } from '@/hooks/useConversations'
import { DrawerContent } from '@/components/DrawerContent'
import { useRouter } from 'expo-router'

function CustomDrawerContent(props) {
  const router = useRouter()
  const {
    conversations,
    activeConversationId,
    isLoading,
    error,
    createConversation,
    switchConversation,
    refetch,
  } = useConversations()

  const handleNewChat = async () => {
    const newId = await createConversation()
    switchConversation(newId)
    router.navigate(`/chat/${newId}`)
    // close drawer
  }

  const handleConversationPress = (id: string) => {
    switchConversation(id)
    router.navigate(`/chat/${id}`)
    // close drawer
  }

  return (
    <DrawerContent
      conversations={conversations}
      activeConversationId={activeConversationId}
      isLoading={isLoading}
      error={error}
      onNewChatPress={handleNewChat}
      onConversationPress={handleConversationPress}
      onRetry={refetch}
    />
  )
}
```
