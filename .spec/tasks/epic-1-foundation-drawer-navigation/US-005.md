## SPECIFICATION

**Objective:** Create a `ConversationActionMenu` component that provides rename and delete actions for a conversation, triggered by long-press. It shows a menu that transitions to either a rename dialog or a delete confirmation alert dialog.

**Success looks like:** In Storybook, users can open the action menu, see rename and delete options, interact with the rename dialog (type a new name and confirm), and interact with the delete confirmation (with a clear destructive warning). All play functions pass.

### Interaction Notes
- The component manages an internal state machine: `idle` -> `menu` -> `rename` | `delete`
- When `open` is true, the initial view is the action menu (rename + delete options)
- Selecting "Rename" transitions to the rename dialog view
- Selecting "Delete" transitions to the delete confirmation AlertDialog view
- Closing any dialog resets back to `idle` state and calls `onOpenChange(false)`
- The rename TextInput must be auto-focused when the rename dialog opens
- The delete button should use destructive styling (red text/background)
- Empty or whitespace-only rename input disables the Save button
