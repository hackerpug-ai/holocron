## CRITICAL CONSTRAINTS

### MUST
- Convert the existing `Stack > (tabs)` routing to `Stack > (drawer) > (tabs)` nesting structure
- Use `expo-router/drawer` (which wraps `@react-navigation/drawer`) for the drawer navigator
- Use the existing `DrawerContent` component from `screens/DrawerContent.tsx` as the custom `drawerContent` prop
- Preserve the existing Storybook bypass in the root `_layout.tsx` (when `EXPO_PUBLIC_STORYBOOK_ENABLED=true`)
- Preserve the existing `+not-found.tsx` and `storybook.tsx` routes at the root level
- Install `react-native-drawer-layout` and `@react-navigation/drawer` if not already present (required by expo-router/drawer)
- Ensure the drawer opens as a pushover (slide-over) overlay, NOT a push-aside drawer

### NEVER
- Remove or break the Storybook toggle logic in `app/_layout.tsx`
- Hardcode hex colors or pixel values; use NativeWind semantic tokens (`bg-background`, `text-foreground`, etc.)
- Use `react-native-paper` Text component; this project uses `@/components/ui/text` from React Native Reusables
- Import from `react-native-drawer-layout` directly in route files; use `expo-router/drawer` abstractions
- Create a new DrawerContent component; wire the existing one from `screens/DrawerContent.tsx`

### STRICTLY
- Follow Expo Router v4/v55 file-based routing conventions: `_layout.tsx` for navigators, group folders `(drawer)` and `(tabs)` for route grouping
- All route files in `app/` directory use default exports (required by Expo Router)
- Named exports only for non-route components outside `app/`
- Every interactive element must have a `testID` prop

## SPECIFICATION

**Objective:** Convert the existing tab-based Expo Router layout to a drawer-based layout so users can access the conversation list, app sections, and navigation via a left-edge pushover drawer, using the already-built `DrawerContent` screen component.

**Success looks like:** App launches and renders the drawer navigator wrapping the existing tab screens. Swiping from the left edge (or tapping a menu icon in the header) opens the `DrawerContent` panel overlaying the main content. Closing the drawer returns to the active tab screen. TypeScript compiles with zero errors. Existing tab navigation (Home, Explore) still works within the drawer context.

### Code Pattern

The drawer layout should follow this Expo Router drawer pattern:

```typescript
// app/(drawer)/_layout.tsx
import { Drawer } from 'expo-router/drawer'
import { DrawerContent } from '@/screens/DrawerContent'

export default function DrawerLayout() {
  return (
    <Drawer
      screenOptions={{
        headerShown: false,
        drawerType: 'slide',
        drawerStyle: { width: '80%' },
      }}
      drawerContent={(props) => <DrawerContent {...props} />}
    >
      <Drawer.Screen name="(tabs)" options={{ headerShown: false }} />
    </Drawer>
  )
}
```

The root layout update:

```typescript
// app/_layout.tsx - change Stack.Screen name
<Stack.Screen name="(drawer)" />  // was: name="(tabs)"
```

### Anti-pattern (DO NOT)

```typescript
// DO NOT create a new drawer content component
// The DrawerContent already exists at screens/DrawerContent.tsx

// DO NOT use push-aside drawer type
drawerType: 'front'  // WRONG: this pushes content aside
// Use: drawerType: 'slide' for pushover behavior

// DO NOT nest drawer inside tabs
// CORRECT: Drawer > Tabs (drawer wraps tabs)
// WRONG: Tabs > Drawer (tabs wrapping drawer)
```
