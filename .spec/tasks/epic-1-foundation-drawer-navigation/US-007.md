## CRITICAL CONSTRAINTS

### MUST
- Wire `ConversationActionMenu` to real Supabase API operations via the `useConversations` hook
- Add `renameConversation` and `deleteConversation` methods to the `useConversations` hook
- Handle the edge case: deleting the active conversation navigates to the next most recent conversation
- Handle the edge case: deleting the last remaining conversation creates a new empty conversation and navigates to it
- Show the `ConversationActionMenu` when a `ConversationRow` is long-pressed in `DrawerContent`
- Validate rename input: trimmed title must not be empty, disallow whitespace-only renames
- Delete must cascade-delete all associated messages (via Supabase foreign key ON DELETE CASCADE)
- Show loading state during rename/delete API operations (disable buttons while in-flight)

### NEVER
- Never delete without user confirmation via the AlertDialog
- Never allow renaming to an empty or whitespace-only string
- Never skip error handling on Supabase mutations -- show error feedback to the user
- Never modify existing UI primitives in `components/ui/`
- Never modify `ConversationRow.tsx` or `DrawerHeader.tsx` -- only wire through `DrawerContent`
- Never use optimistic updates without rollback -- if the mutation fails, revert the UI state

### STRICTLY
- Rename uses `supabaseAdmin.from('conversations').update({ title }).eq('id', conversationId)`
- Delete uses `supabaseAdmin.from('conversations').delete().eq('id', conversationId)`
- After successful rename or delete, the conversation list must be refetched to reflect the change
- The `ConversationActionMenu` must receive the selected conversation's ID and title as props
- All new hook methods must have TypeScript return types and error handling

## SPECIFICATION

**Objective:** Wire the `ConversationActionMenu` component to real Supabase rename and delete operations, handling all edge cases for active conversation deletion and last-conversation deletion, integrated into the drawer through the `useConversations` hook.

**Success looks like:** User can long-press any conversation in the drawer, see a menu with Rename and Delete options. Renaming updates the title in Supabase and reflects immediately in the drawer list. Deleting removes the conversation and all its messages from Supabase. If the active conversation is deleted, the app navigates to the next most recent. If the last conversation is deleted, a new empty one is created automatically.

### Interaction Notes
- Long-press on ConversationRow triggers `onConversationLongPress(conversationId)` in DrawerContent
- DrawerContent manages state for which conversation is selected for action and whether the menu is open
- `ConversationActionMenu` receives the selected conversation's title and callbacks
- On rename confirm: call `renameConversation(id, newTitle)` from the hook
- On delete confirm: call `deleteConversation(id)` from the hook
- After delete of active conversation: hook returns updated list, drawer layout navigates to most recent
- After delete of last conversation: hook creates new conversation, returns it as the only item

### Code Pattern
```typescript
// hooks/useConversations.ts -- additions
export function useConversations(): UseConversationsReturn {
  // ... existing code from US-006 ...

  const renameConversation = async (id: string, newTitle: string): Promise<void> => {
    const trimmed = newTitle.trim()
    if (!trimmed) throw new Error('Title cannot be empty')

    const { error } = await supabaseAdmin
      .from('conversations')
      .update({ title: trimmed, updated_at: new Date().toISOString() })
      .eq('id', id)

    if (error) throw error
    await fetchConversations() // refetch to update list
  }

  const deleteConversation = async (id: string): Promise<string | null> => {
    const { error } = await supabaseAdmin
      .from('conversations')
      .delete()
      .eq('id', id)

    if (error) throw error

    // If deleting active conversation, determine next active
    if (id === activeConversationId) {
      const remaining = conversations.filter(c => c.id !== id)
      if (remaining.length === 0) {
        // Create new conversation if last one deleted
        const newId = await createConversation()
        return newId // caller navigates to this
      }
      // Return most recent remaining
      const next = remaining[0]
      setActiveConversationId(next.id)
      await fetchConversations()
      return next.id
    }

    await fetchConversations()
    return null // no navigation needed
  }

  return {
    // ... existing returns ...
    renameConversation,
    deleteConversation,
  }
}
```

```typescript
// components/DrawerContent.tsx -- additions for long-press wiring
export function DrawerContent({ ... }: DrawerContentProps) {
  const [actionMenuConversation, setActionMenuConversation] = useState<{
    id: string
    title: string
  } | null>(null)

  const handleLongPress = (id: string) => {
    const conv = conversations.find(c => c.id === id)
    if (conv) {
      setActionMenuConversation({ id: conv.id, title: conv.title })
    }
  }

  return (
    <View ...>
      {/* ... existing drawer content ... */}

      <ConversationActionMenu
        open={actionMenuConversation !== null}
        onOpenChange={(open) => {
          if (!open) setActionMenuConversation(null)
        }}
        conversationTitle={actionMenuConversation?.title ?? ''}
        onRename={(newTitle) => onRename?.(actionMenuConversation!.id, newTitle)}
        onDelete={() => onDelete?.(actionMenuConversation!.id)}
      />
    </View>
  )
}
```

### Anti-pattern (DO NOT)
```typescript
// DO NOT: Delete without confirmation
const handleDelete = async (id: string) => {
  await supabaseAdmin.from('conversations').delete().eq('id', id) // BAD - no confirmation
}

// DO NOT: Use optimistic updates without rollback
const handleRename = (id: string, title: string) => {
  setConversations(prev => prev.map(c => c.id === id ? { ...c, title } : c))
  // If the API call fails, the UI shows stale data
  supabaseAdmin.from('conversations').update({ title }).eq('id', id) // fire and forget BAD
}

// DO NOT: Ignore the "delete last conversation" edge case
const handleDelete = async (id: string) => {
  await deleteFromDB(id)
  // BAD: if this was the last conversation, user sees empty state with no way forward
}
```

## NOTES
- The ON DELETE CASCADE foreign key on the `messages` table handles message cleanup automatically when a conversation is deleted. Verify this constraint exists in the Supabase schema.
- Rename does not trigger an embedding update (unlike document title changes in `lib/supabase.ts`) because conversations don't have embeddings.
- The `deleteConversation` method returns a `string | null` indicating the conversation ID to navigate to (if the active conversation was deleted) or `null` (if no navigation is needed). The drawer layout handles the actual navigation.
- Consider adding a brief toast or snackbar notification after successful rename/delete for user feedback, but this can be deferred if no toast component exists yet.
