## CRITICAL CONSTRAINTS

### MUST
- Compose existing `DrawerHeader` (from `components/DrawerHeader.tsx`) and `ConversationRow` (from `components/ConversationRow.tsx`) components -- do NOT reimplement them
- Use NativeWind utility classes for all styling (no inline style objects for static layout)
- Use `cn()` from `@/lib/utils` for conditional class merging
- Import `Text` from `@/components/ui/text` -- never from `react-native`
- Include `testID` on every interactive and structural element
- Co-locate a Storybook story file (`DrawerContent.stories.tsx`) as a sibling
- Use mock data for all stories -- no API calls, no hooks that fetch data
- Support ScrollView for the conversation list to handle 15+ conversations

### NEVER
- Never hardcode hex colors -- use NativeWind semantic tokens (`bg-background`, `text-foreground`, `bg-muted`, etc.)
- Never hardcode spacing values -- use Tailwind spacing classes (`p-3`, `gap-2`, `mt-1`, etc.)
- Never import `Text` from `react-native` -- always use `@/components/ui/text`
- Never add API fetching logic -- this is a design-only task with mock data
- Never modify existing `DrawerHeader.tsx` or `ConversationRow.tsx` components
- Never use `useCallback` or `useMemo` unless meeting the specific criteria in CLAUDE.md

### STRICTLY
- All props must be TypeScript interfaces with JSDoc comments
- Event handlers must follow `on{Event}` naming convention
- Component file must be named `DrawerContent.tsx` and export a named function `DrawerContent`
- Story file must follow the `Meta<typeof DrawerContent>` pattern from existing stories
- The component is the FULL drawer panel content: header + section divider + "Conversations" label + scrollable list

## SPECIFICATION

**Objective:** Create a `DrawerContent` composite component that assembles `DrawerHeader` and a scrollable list of `ConversationRow` items into the complete drawer panel, following the ChatGPT-style drawer pattern.

**Success looks like:** A fully rendered drawer panel in Storybook showing the header (search bar + compose + section links), a "Conversations" label divider, and a scrollable list of conversation rows with active state highlighting. All four story variants render without errors and the play function passes.

### Interaction Notes
- The component receives a flat array of conversation data and maps to `ConversationRow` components
- `onConversationPress(conversationId)` -- called when a row is tapped
- `onConversationLongPress(conversationId)` -- called when a row is long-pressed
- `onNewChatPress()` -- passed through to DrawerHeader
- `onSearchChange(query)` -- passed through to DrawerHeader (placeholder only, no filtering)
- Active conversation is identified by `activeConversationId` prop matching a conversation's `id`
- Empty state renders a centered message when conversations array is empty

### Code Pattern
```typescript
// Composition pattern: DrawerContent composes DrawerHeader + ConversationRow list
import { DrawerHeader } from '@/components/DrawerHeader'
import { ConversationRow } from '@/components/ConversationRow'
import { Text } from '@/components/ui/text'
import { ScrollView, View } from 'react-native'
import { cn } from '@/lib/utils'

export interface Conversation {
  id: string
  title: string
  lastMessage?: string
  lastMessageAt?: Date
}

interface DrawerContentProps {
  conversations: Conversation[]
  activeConversationId?: string
  onConversationPress?: (id: string) => void
  onConversationLongPress?: (id: string) => void
  onNewChatPress?: () => void
  onSearchChange?: (query: string) => void
  searchQuery?: string
}

export function DrawerContent({ ... }: DrawerContentProps) {
  return (
    <View className="bg-background flex-1" testID="drawer-content">
      <DrawerHeader ... />
      {/* Divider */}
      {/* "Conversations" label */}
      <ScrollView>
        {conversations.length === 0 ? (
          <EmptyState />
        ) : (
          conversations.map(conv => <ConversationRow key={conv.id} ... />)
        )}
      </ScrollView>
    </View>
  )
}
```
