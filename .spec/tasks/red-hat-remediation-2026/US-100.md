- No service role references in client codebase

## EXAMPLES

**Follow this exact pattern for anon key client:**

```typescript
// lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = process.env.EXPO_PUBLIC_SUPABASE_URL!
const supabaseAnonKey = process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY!

export const supabase = createClient(supabaseUrl, supabaseAnonKey)
```

**Why this pattern:**
- Anon key is safe for client distribution
- Designed for public-facing clients with RLS protection
- Cannot bypass Row Level Security policies

**Anti-pattern (DO NOT do this):**

```typescript
// ❌ SECURITY VULNERABILITY - NEVER DO THIS
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = process.env.EXPO_PUBLIC_SUPABASE_URL!
const supabaseServiceKey = process.env.EXPO_PUBLIC_SUPABASE_SERVICE_ROLE_KEY! // ❌ BUNDLES SECRET INTO APP

export const supabaseAdmin = createClient(supabaseUrl, supabaseServiceKey)
```
