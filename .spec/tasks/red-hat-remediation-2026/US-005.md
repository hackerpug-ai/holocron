## EXAMPLES

**Follow this exact pattern for Paper Menu:**

```typescript
// components/ConversationActionMenu.tsx
import React, { useState } from 'react'
import { Menu } from 'react-native-paper'

interface ConversationActionMenuProps {
  visible: boolean
  onDismiss: () => void
  onRename: () => void
  onDelete: () => void
  anchor: { x: number; y: number }
}

export function ConversationActionMenu({
  visible,
  onDismiss,
  onRename,
  onDelete,
  anchor,
}: ConversationActionMenuProps) {
  return (
    <Menu
      visible={visible}
      onDismiss={onDismiss}
      anchor={anchor}
      contentStyle={{
        minWidth: 150,
      }}
    >
      <Menu.Item
        leadingIcon="pencil"
        onPress={() => {
          onRename()
          onDismiss()
        }}
        title="Rename"
      />
      <Menu.Item
        leadingIcon="delete"
        onPress={() => {
          onDelete()
          onDismiss()
        }}
        title="Delete"
      />
    </Menu>
  )
}
```

**Why this pattern:**
- Uses react-native-paper Menu (standard component)
- Passes anchor position for menu placement
- Provides callbacks for rename/delete actions
- Auto-dismisses after action selection

**Anti-pattern (DO NOT do this):**

```typescript
// ❌ CUSTOM MENU IMPLEMENTATION
// Don't build your own menu with View/Pressable
export function BadMenu() {
  return (
    <View style={styles.menuContainer}>
      {/* Custom menu implementation - use Paper instead! */}
    </View>
  )
}
```
