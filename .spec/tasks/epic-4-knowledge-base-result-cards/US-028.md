# Build /stats command handler

> Task ID: US-028
> Type: feature
> Priority: P2
> Appetite Fit: small (fits)
> Assignee: supabase-implementer
> PRD Version: 3.1.0
> Stability: evolving

## CRITICAL CONSTRAINTS

### MUST
- Handle /stats command in chat-send Edge Function
- Return stats card with total document count
- Include breakdown of documents by category
- Show 5 most recent documents as tappable links

### NEVER
- Return stats as plain text (must use card_data)
- Include more than 5 recent documents
- Cache stats (always compute fresh)

### STRICTLY
- Use message_type='result_card' and card_type='stats'

## SPECIFICATION

**Objective:** Implement the /stats command handler that returns knowledge base statistics as a summary card in the chat.

**Success looks like:** Submitting '/stats' returns a stats card showing total documents, category breakdown, and 5 most recent documents.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | User submits '/stats' | chat-send processes /stats command | Agent responds with stats card | `pnpm supabase functions invoke chat-send --body '{"conversation_id":"test","content":"/stats","message_type":"text"}'` |
| 2 | Stats card is returned | Response is rendered | Card shows total document count | Manual inspection |
| 3 | Stats card is returned | Response is rendered | Card shows category breakdown with counts | Manual inspection |
| 4 | Stats card is returned | Response is rendered | Card shows 5 most recent documents | Manual inspection |
