# Wire ResultCard to chat-send response rendering

> Task ID: US-030
> Type: feature:integration
> Priority: P2
> Appetite Fit: medium (fits)

> PRD Version: 3.1.0

## CRITICAL CONSTRAINTS

### MUST
- Render ResultCard in MessageBubble when message_type='result_card'
- Pass card_data from chat message to ResultCard component
- Handle all card_type variants (article, stats, category_list, etc.)
- Wire card onPress to navigate to ArticleDetail or handle actions
- Handle loading and error states for card interactions

### NEVER
- Create new message rendering logic (extend existing MessageBubble)
- Hardcode card data (use from message.card_data)

### STRICTLY
- Follow MessageBubble pattern for conditional rendering
- Preserve scroll position when card tapped
- Handle null/undefined card_data gracefully

## SPECIFICATION

**Objective:** Integrate ResultCard component into the chat thread so result_card messages display as interactive cards.

**Success looks like:** Result cards from /search, /browse, /stats commands render correctly in the chat thread with tappable actions.

### Code Pattern

```typescript
// components/chat/MessageBubble.tsx
import { ResultCard } from '@/components/ui/result-card'
import { useArticleNavigation } from '@/hooks/use-article-navigation'

export function MessageBubble({ message }) {
  const articleNav = useArticleNavigation()

  if (message.message_type === 'result_card' && message.card_data) {
    return (
      <View className="my-2">
        {Array.isArray(message.card_data) ? (
          // Multiple cards (search results)
          message.card_data.map((card, index) => (
            <ResultCard
              key={index}
              cardType={card.card_type}
              data={card}
              onPress={() => articleNav.openArticle(card.document_id)}
              testID={`chat-result-card-${index}`}
            />
          ))
        ) : (
          // Single card (stats, category_list, etc.)
          <ResultCard
            cardType={message.card_data.card_type}
            data={message.card_data}
            testID="chat-result-card"
          />
        )}
      </View>
    )
  }

  // ... other message types
}
```
