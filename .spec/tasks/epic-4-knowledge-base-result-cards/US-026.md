# Build /search command handler with hybrid_search

> Task ID: US-026
> Type: feature
> Priority: P1
> Appetite Fit: medium (fits)
> Assignee: supabase-implementer
> PRD Version: 3.1.0
> Stability: evolving

## CRITICAL CONSTRAINTS

### MUST
- Handle /search command in chat-send Edge Function
- Call existing hybrid_search RPC function with query text
- Return results as result_card message_type with card_data payload
- Return "no results found" card when search returns empty
- Include title, category, snippet, and document_id in card_data

### NEVER
- Create new search RPC (use existing hybrid_search)
- Return results as plain text (must use card_data)
- Implement query refinement (deferred to future)
- Fail silently - always return a card (results or empty)

### STRICTLY
- Use existing rpc/hybrid_search function
- Limit results to 10 documents per search
- Store message_type='result_card' and card_data with card_type='article'

## SPECIFICATION

**Objective:** Implement the /search command handler that queries the holocron knowledge base using the existing hybrid_search RPC function and returns results as tappable result cards in the chat stream.

**Success looks like:** Submitting '/search React hooks' returns an agent message with a result_card containing article results with title, category badge, snippet, and tappable action.

### Result Card Schema

```typescript
// card_data structure for article result cards
interface ArticleResultCard {
  card_type: 'article'
  title: string
  category: string  // document_category enum
  snippet: string   // First 150 chars
  document_id: number
  metadata?: {
    relevance_score?: number
    date?: string
  }
}

// Message structure
{
  role: 'agent',
  content: 'Found 5 articles matching "React hooks"',
  message_type: 'result_card',
  card_data: ArticleResultCard[] | { card_type: 'no_results', message: string }
}
```

### Code Pattern

```typescript
// In supabase/functions/chat-send/index.ts
import { parseSlashCommand, isKnownCommand } from './slash-commands.ts'

// After inserting user message, check for /search
const parsed = parseSlashCommand(body.content)

let agentContent: string
let agentMessageType: MessageType = 'text'
let agentCardData: CardData | undefined

if (parsed.isCommand && parsed.command === 'search') {
  // Extract query (after /search or from natural language)
  const query = parsed.args || body.content

  // Call existing hybrid_search RPC
  const { data: searchResults, error: searchError } = await supabase.rpc('hybrid_search', {
    query_text: query,
    match_threshold: 0.5,
    result_count: 10
  })

  if (searchError || !searchResults || searchResults.length === 0) {
    // No results found
    agentContent = 'No results found'
    agentMessageType = 'result_card'
    agentCardData = {
      card_type: 'no_results',
      message: `No articles found matching "${query}"`
    }
  } else {
    // Results found - build card data
    agentContent = `Found ${searchResults.length} articles matching "${query}"`
    agentMessageType = 'result_card'
    agentCardData = searchResults.map((doc: any) => ({
      card_type: 'article',
      title: doc.title,
      category: doc.category,
      snippet: doc.content?.substring(0, 150) + '...',
      document_id: doc.id,
      metadata: {
        relevance_score: doc.similarity || doc.rank
      }
    }))
  }
} else {
  // ... other command handling
}

// Insert agent message
const { data: agentMessage, error: agentError } = await supabase
  .from('chat_messages')
  .insert({
    conversation_id: body.conversation_id,
    role: 'agent' as MessageRole,
    content: agentContent,
    message_type: agentMessageType,
    card_data: agentCardData
  })
  .select()
  .single()
```
