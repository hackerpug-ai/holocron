# Implement command insertion with syntax hint

> Task ID: US-024
> Type: feature:integration
> Priority: P2

> Assignee: react-native-ui-implementer
> PRD Version: 3.1.0

## CRITICAL CONSTRAINTS

### MUST
- When user selects command, insert full command into input
- Append syntax hint after command (e.g., '/research <question>')
- Position cursor after command for arg entry
- Hide command panel after selection
- Clear syntax hint when user starts typing args

### NEVER
- Validate arguments before submit (deferred per deliberation)
- Auto-submit on command selection
- Keep panel visible after selection

### STRICTLY
- Use command.syntax from SlashCommand interface
- Syntax hint displayed in muted color
- Hint disappears when user types

## SPECIFICATION

**Objective:** Implement the command selection flow where tapping a command inserts it into the input with its syntax hint, allowing the user to type arguments and submit.

**Success looks like:** Selecting '/research' inserts '/research <question>' into input, hint is muted, and typing clears the hint portion.

### Code Pattern

```typescript
// In ChatInput.tsx - enhanced command selection handling
import type { SlashCommand } from '@/components/SlashCommandMenu'

// Add state for syntax hint
const [syntaxHint, setSyntaxHint] = useState<string | null>(null)
const [selectedCommand, setSelectedCommand] = useState<string | null>(null)

// Handle command selection
const handleCommandSelect = (command: SlashCommand) => {
  const newValue = `/${command.name} `
  setValue(newValue)
  setShowCommandPanel(false)
  setSelectedCommand(command.name)

  // Store syntax hint for placeholder display
  if (command.syntax) {
    setSyntaxHint(command.syntax)
  } else {
    setSyntaxHint(null)
  }
}

// Clear hint when user types after command
const handleTextChange = (text: string) => {
  setValue(text)

  // Clear hint if user started typing args
  if (syntaxHint && selectedCommand) {
    const commandPrefix = `/${selectedCommand} `
    if (text.length > commandPrefix.length) {
      setSyntaxHint(null)
      setSelectedCommand(null)
    }
  }

  // Reset if user clears the command
  if (!text.startsWith('/')) {
    setSyntaxHint(null)
    setSelectedCommand(null)
  }

  setShowCommandPanel(text.startsWith('/'))
}

// Dynamic placeholder
const dynamicPlaceholder = syntaxHint ? syntaxHint : placeholder

// In JSX
<TextInput
  testID="chat-input-field"
  className="flex-1 bg-muted rounded-full px-4 py-2 text-foreground"
  placeholder={dynamicPlaceholder}
  placeholderTextColor={syntaxHint ? themeColors.primary : themeColors.mutedForeground}
  value={value}
  onChangeText={handleTextChange}
  onSubmitEditing={handleSend}
  editable={!disabled}
/>
```

### Alternative Pattern: Suffix Hint

```typescript
// If placeholder approach doesn't work well, use a suffix Text
<View className="flex-row items-center flex-1">
  <TextInput
    value={value}
    onChangeText={handleTextChange}
    className="flex-1"
  />
  {syntaxHint && value === `/${selectedCommand} ` && (
    <Text className="text-muted-foreground text-sm absolute right-0">
      {syntaxHint}
    </Text>
  )}
</View>
```
