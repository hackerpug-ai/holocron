# Red-Hat Review Report

**Report Date**: 2026-03-23
**Target**: Chat Agent + Audio Narration Features
**Reviewed By**: convex-reviewer, code-reviewer (frontend), security-reviewer

## Executive Summary

The chat agent and audio narration features have significant functional gaps that prevent them from working end-to-end. The most critical: tool approval cards are invisible (toolCallId never reaches the UI), the regenerate button is entirely non-functional, paragraph indices are mismatched between frontend/backend causing wrong highlights, and audio duration is always 0. The chat agent's tool stubs return empty data to the LLM, and `agentBusy` can permanently lock a conversation.

---

## HIGH Confidence Findings (3+ Agents Agree)

- [ ] **Tool approval cards are invisible**: `toolCallId` is never mapped through `useChatHistory` to the UI. `MessageBubble` checks `toolCallId` but it's always `undefined`. The human-in-the-loop flow is completely broken. | Severity: **Critical**
  - Agents: convex-reviewer, code-reviewer

- [ ] **`agentBusy` can be permanently stuck `true`**: No TTL, no watchdog, no cancel button. If a Convex action crashes/times out before the `finally` block, the conversation is permanently locked with a disabled input. | Severity: **Critical**
  - Agents: convex-reviewer, code-reviewer, security-reviewer

- [ ] **Regenerate button is non-functional**: `NarrationControlBar` calls `narration.regenerate()` which only changes local state to `'generating'`. Neither integration file has a `useEffect` to call `regenerateForDocument` when state enters `'generating'`. The `regenerateAction` is imported but never called. | Severity: **Critical**
  - Agents: convex-reviewer, code-reviewer

- [ ] **Paragraph index mismatch between frontend and backend**: Frontend splits on `\n\n+`, backend's `extractParagraphs` also strips code blocks, images, HTML, and treats headings as separate segments. Active paragraph highlight will be shifted from actual audio playback. | Severity: **High**
  - Agents: convex-reviewer, code-reviewer

- [ ] **`durationMs` always stored as 0**: ElevenLabs duration is never computed. `totalTimeSeconds` is always 0, progress bar never moves, time displays show `0:00`. | Severity: **High**
  - Agents: convex-reviewer, code-reviewer

- [ ] **`onParagraphReady` / `onAllReady` never fire from segments subscription**: No `useEffect` watches `segments` to dispatch `PARAGRAPH_READY` or `ALL_READY` as segments complete. `generatedCount` stays 0. | Severity: **High**
  - Agents: convex-reviewer, code-reviewer

- [ ] **Tool `execute` stubs all return empty data**: All 13 tools in `tools.ts` have `execute` functions returning hardcoded empty results. The LLM sees empty data and generates responses based on it. The real `toolExecutor.ts` implementations are disconnected from the AI SDK's tool execution path. | Severity: **High**
  - Agents: convex-reviewer, code-reviewer

- [ ] **No rate limiting on audio generation**: `regenerateForDocument` has no cooldown. Can be called in a tight loop to exhaust ElevenLabs API credits. | Severity: **High**
  - Agents: convex-reviewer, security-reviewer

- [ ] **Narration gets stuck in `generating` on any error**: `handleToggleNarration` has no try/catch. If `generateAction` throws, state machine stays in `'generating'` forever with spinning ActivityIndicator and no user feedback. | Severity: **High**
  - Agents: code-reviewer, security-reviewer

## Recommendations by Category

### 1. Gaps (Must Fix for Feature to Work)
1. **Wire `toolCallId` through `useChatHistory` → `ChatThread` → `MessageBubble`** so tool approval cards render
2. **Add `useEffect` watching `segments`** to call `onParagraphReady`/`onAllReady` as segments complete
3. **Wire `regenerateAction`** to a `useEffect` that fires when state enters `'generating'` from a regenerate action (not initial enter)
4. **Align paragraph counting** between frontend split and backend `extractParagraphs`
5. **Compute audio duration** from the buffer length in `generateSegment` (buffer.length / sampleRate / channels)

### 2. Risks (Should Fix)
6. **Add try/catch to `handleToggleNarration`** with error state in the reducer
7. **Add TTL/watchdog for `agentBusy`** — auto-reset after 5 minutes
8. **Add iteration depth limit** to `continueAfterTool` (e.g., max 10 tool calls per turn)
9. **Move `ctx.storage.delete`** out of `deleteAllForDocument` mutation into an action
10. **Add rate limiting** for `regenerateForDocument` (per-document cooldown)

### 3. Assumptions (Should Validate)
11. Verify `toolCalls.mutations.create` is `internalMutation` not public
12. Test rapid paragraph skipping on device to validate Sound lifecycle
13. Confirm `Audio.Sound.createAsync` honors cancellation pattern correctly

### 4. Polish (Nice to Have)
14. Make `handleRetry` actually retry the last message
15. Add error state UI for narration generation failure
16. Fix `NarrationToggleButton` icon semantics (Mic when active, MicOff when inactive)
