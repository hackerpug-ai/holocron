### Interaction Notes

- Hybrid storage: Large transcript content goes in Convex file storage, metadata in database
- Idempotent job creation: `by_content` index prevents duplicate jobs for same video
- Job queue processing: `by_priority` index enables high-priority job selection
- Status machine: Union types ensure only valid status transitions

### Code Pattern

Source: `convex/schema.ts` - audioSegments table (existing)

```typescript
audioSegments: defineTable({
  storageId: v.id("_storage"),
  recordingId: v.id("audioRecordings"),
  segmentIndex: v.number(),
  startTime: v.number(),
  endTime: v.number(),
  duration: v.number(),
  transcript: v.optional(v.string()),
  embeddingId: v.optional(v.id("embeddings")),
  createdAt: v.number(),
})
.index("by_recording", ["recordingId"]),
```

Apply this pattern for videoTranscripts:

```typescript
videoTranscripts: defineTable({
  contentId: v.string(), // YouTube video ID
  sourceUrl: v.string(), // YouTube video URL
  transcriptType: v.union(v.literal("api"), v.literal("generated"), v.literal("jina_fallback")),
  transcriptSource: v.string(), // youtube_api, jina_reader_api
  storageId: v.id("_storage"), // Full transcript in file storage
  previewText: v.string(), // First 500 chars for search/display
  wordCount: v.number(),
  durationMs: v.optional(v.number()),
  language: v.optional(v.string()),
  metadataJson: v.optional(v.any()),
  generatedAt: v.number(),
  createdAt: v.number(),
})
.index("by_content_id", ["contentId"])
.index("by_source_url", ["sourceUrl"]),
```

### Anti-pattern (DO NOT)

```typescript
// DON'T: Store full transcript in database field
videoTranscripts: defineTable({
  contentId: v.string(),
  fullTranscript: v.string(), // WRONG - too large, use file storage
  // ...
})
```

## CODING STANDARDS

- **brain/docs/coding-standards/core-typescript-rules.md**:
  - Use `v.union()` for enum-like fields (transcriptType, status)
  - Use `v.optional()` for nullable fields (durationMs, language, metadataJson)
  - Always include createdAt timestamp for ordering
