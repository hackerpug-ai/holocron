# US-005: Implement useVoiceSessionState reducer for voice state machine

> Task ID: US-005
> Type: FEATURE
> Priority: P0
> Estimate: 45 minutes
> Assignee: react-native-ui-implementer

## CRITICAL CONSTRAINTS

### MUST
- Implement as useReducer with states: IDLE, CONNECTING, LISTENING, PROCESSING, SPEAKING, ERROR
- Support actions: CONNECT, CONNECTED, START_LISTENING, START_SPEAKING, STOP_SPEAKING, ERROR, DISCONNECT, TIMEOUT
- Include sessionId, conversationId, and error message in state

### NEVER
- Store voice state in Convex — it's client-only
- Use useState for coordinated state changes
- Allow invalid state transitions (e.g., IDLE -> SPEAKING)

### STRICTLY
- Export reducer function separately for unit testing
- Follow useReducer pattern per CLAUDE.md React rules

## SPECIFICATION

**Objective:** Create a useReducer-based voice session state machine that manages all voice UI states

**Success looks like:** Reducer correctly transitions between all valid states, rejects invalid transitions, and carries session metadata
