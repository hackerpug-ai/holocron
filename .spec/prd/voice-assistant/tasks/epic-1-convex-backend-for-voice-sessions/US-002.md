## CRITICAL CONSTRAINTS

### MUST
- POST to https://api.openai.com/v1/realtime/client_secrets (NOT /v1/realtime/sessions which is deprecated)
- Use OPENAI_API_KEY from Convex environment variables, never expose to client
- Return ephemeral key, expiresAt, and sessionId to client

### NEVER
- Return the OPENAI_API_KEY to the client
- Store the ephemeral key in the database
- Allow creating a session if one is already active for the conversationId

### STRICTLY
- Use 'use node' directive for HTTP fetch in Convex action
- Create session record via internal mutation, not direct db access in action
- Follow convex/audio/actions.ts pattern for action structure

## SPECIFICATION

**Objective:** Implement a Convex action that generates an OpenAI Realtime ephemeral token and creates a voiceSession record

**Success looks like:** Client can call voice.createSession with a conversationId and receive {ephemeralKey, expiresAt, sessionId}

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | A valid conversationId and OPENAI_API_KEY in env | voice.createSession is called | Returns object with ephemeralKey (string starting with 'ek_'), expiresAt (number), sessionId (Id<voiceSessions>) | `pnpm vitest run --reporter=verbose 2>&1 \| grep createSession` |
| 2 | A conversationId that already has an active session (completedAt is undefined) | voice.createSession is called with same conversationId | Throws error indicating active session already exists | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'active session'` |
| 3 | OPENAI_API_KEY is missing or invalid | voice.createSession is called | Throws descriptive error without leaking key details | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'API key'` |
| 4 | OpenAI API returns non-200 response | voice.createSession is called | Throws error with status code context, no session record created | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'non-200'` |
