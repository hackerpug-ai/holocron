## CRITICAL CONSTRAINTS

### MUST
- Detect WebRTC connection failure and data channel close events
- Retry with exponential backoff: 1s, 2s, 4s (max 3 retries)
- Provide spoken feedback at each retry via data channel or local TTS fallback

### NEVER
- Retry indefinitely — max 3 attempts
- Leave orphaned RTCPeerConnection on failure
- Show technical error details to user

### STRICTLY
- Use connection state change events (iceconnectionstatechange, connectionstatechange)
- On retry exhaustion, transition to ERROR state with user-friendly message

## SPECIFICATION

**Objective:** Add network failure detection, automatic retry with backoff, and spoken error feedback to the voice session

**Success looks like:** Network failures trigger automatic retry with spoken status updates; after max retries, session ends with clear message

```typescript
export class RetryManager {
  async retry<T>(
    fn: () => Promise<T>,
    maxRetries = 3,
    baseDelay = 1000
  ): Promise<T> {
    for (let i = 0; i < maxRetries; i++) {
      try {
        return await fn();
      } catch (error) {
        if (i === maxRetries - 1) throw error;
        await new Promise(r => setTimeout(r, baseDelay * Math.pow(2, i)));
      }
    }
    throw new Error('Unreachable');
  }
}
```
