# US-017: Implement session timeout behavior with spoken farewell

> Task ID: US-017
> Type: FEATURE
> Priority: P0
> Estimate: 60 minutes
> Assignee: ai-tooling-implementer

## CRITICAL CONSTRAINTS

### MUST
- Use OpenAI's idle_timeout_ms (30000ms) in server_vad config for timeout detection
- On timeout, transition to IDLE with gentle farewell
- Keep WebRTC connection warm for 5 minutes for quick re-activation

### NEVER
- Abruptly close connection on timeout — speak farewell first
- Destroy warm connection resources during the 5-minute window

### STRICTLY
- Timeout is interruptible — if user speaks during timeout announcement, cancel timeout
- Quick re-activation after timeout must be <200ms

## SPECIFICATION

**Objective:** Implement session timeout with gentle spoken farewell and warm connection for quick re-activation

**Success looks like:** 30s silence triggers spoken farewell and idle transition; warm connection enables instant re-activation for 5 minutes

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | Active voice session in LISTENING state | 30 seconds of silence pass (no speech detected) | Assistant speaks 'I will be here when you need me', state transitions to IDLE | `pnpm vitest run --reporter=verbose 2>&1 \| grep timeout` |
| 2 | Timeout farewell is being spoken | User speaks during the farewell | Timeout cancelled, session resumes LISTENING, user input processed | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'timeout interrupted'` |
| 3 | Session timed out within last 5 minutes | User taps mic button to re-activate | Session resumes within 200ms using warm connection (no new ephemeral token needed) | `pnpm vitest run --reporter=verbose 2>&1 \| grep 're-activation'` |
| 4 | Session timed out more than 5 minutes ago | User taps mic button | Full new session created (new ephemeral token, new WebRTC connection) | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'cold start'` |

```typescript
// Session config includes timeout
const sessionConfig = {
  turn_detection: {
    type: 'server_vad',
    threshold: 0.5,
    prefix_padding_ms: 300,
    silence_duration_ms: 500,
    idle_timeout_ms: 30000,  // 30s timeout
    create_response: true,
    interrupt_response: true,
  }
};

// Warm connection manager
class WarmConnection {
  private timeoutId: NodeJS.Timeout | null = null;
  private connection: WebRTCConnection | null = null;
  private readonly WARM_DURATION = 5 * 60 * 1000; // 5 minutes

  startWarmPeriod(conn: WebRTCConnection) {
    this.connection = conn;
    this.timeoutId = setTimeout(() => this.destroy(), this.WARM_DURATION);
  }

  reactivate(): WebRTCConnection | null {
    if (this.connection && this.timeoutId) {
      clearTimeout(this.timeoutId);
      return this.connection;
    }
    return null; // cold start needed
  }
}
```

### Anti-pattern (DO NOT)
Do NOT close WebRTC connection immediately on timeout. Keep warm for 5 minutes.
