> Type: FEATURE
> Priority: P0

## CRITICAL CONSTRAINTS

### MUST
- Define all P0 tools: search_knowledge, list_recent_documents, get_document, get_conversations, get_research_sessions, get_improvements, check_agent_status, start_research, submit_improvement, create_note, navigate_app
- Each tool has name, description, and JSON Schema parameters
- Tools registered via session.update sent on data channel after connection

### NEVER
- Hardcode tool definitions inline — define in a separate config module
- Include tools that require P1/P2 features
- Exceed 16,384 tokens for instructions + tools combined

### STRICTLY
- Follow exact tool definition format from 03-function-calling.md
- Tool names must match exactly what the function dispatcher expects

## SPECIFICATION

**Objective:** Create tool definition config and integrate with session.update to register tools with OpenAI Realtime

**Success looks like:** All P0 tools registered with OpenAI at session start; tool definitions are typed and maintainable
