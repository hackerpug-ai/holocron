# US-010: Implement function call dispatcher that routes to Convex endpoints

> Task ID: US-010
> Type: FEATURE
> Priority: P0
> Estimate: 90 minutes
> Assignee: ai-tooling-implementer

## CRITICAL CONSTRAINTS

### MUST
- Route function calls by name to correct Convex query/mutation/action
- Return results via conversation.item.create with correct call_id, then response.create
- Handle async agent dispatchers (start_research, submit_improvement) with timeout

### NEVER
- Use item.id as call_id — must use item.call_id (format: call_XXXX)
- Send response.create while a response is already active (causes conversation_already_has_active_response error)
- Skip the response.create after conversation.item.create (model won't respond)

### STRICTLY
- Pure reads execute immediately; agent dispatchers use timeout pattern from PRD
- All results JSON.stringified before sending as output
- Record each function call via voice.recordCommand mutation

## SPECIFICATION

**Objective:** Create a dispatcher that receives function call events from OpenAI, executes them against Convex, returns results on the data channel, and records execution audit trail

**Success looks like:** All P0 tools execute correctly, results returned to OpenAI for narration, executions recorded in voiceCommands

### Code Pattern
Source: .spec/prd/voice-assistant/technical-requirements/03-function-calling.md:54-70

```typescript
executeTool(item.name, JSON.parse(item.arguments)).then(output => {
  dc.send(JSON.stringify({
    type: "conversation.item.create",
    item: {
      type: "function_call_output",
      call_id: item.call_id,  // MUST match
      output: JSON.stringify(output)
    }
  }));
  dc.send(JSON.stringify({ type: "response.create" }));
});
```

### Anti-pattern (DO NOT)
Do NOT use item.id as call_id. Do NOT forget response.create after returning result.
