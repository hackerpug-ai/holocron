# US-011: Implement transcript persistence — record voice turns to chatMessages

> Task ID: US-011
> Type: FEATURE
> Priority: P0
> Estimate: 45 minutes
> Assignee: ai-tooling-implementer

## CRITICAL CONSTRAINTS

### MUST
- Listen for response.audio_transcript.done events for assistant transcripts
- Listen for input_audio_transcription events for user transcripts
- Call voice.recordTranscript for each completed turn

### NEVER
- Record partial transcripts as final messages
- Skip recording assistant responses — both user and agent turns must be persisted
- Block audio playback while recording transcript

### STRICTLY
- Integrate with useVoiceSession hook as a side-effect module
- Transcript recording is fire-and-forget (don't await before continuing)

## SPECIFICATION

**Objective:** Wire data channel transcript events to Convex recordTranscript mutation so voice conversations appear in chat history

**Success looks like:** All voice turns (user and assistant) appear in the chat conversation thread as text messages

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | An active voice session with data channel events flowing | User speaks and input_audio_transcription event received | voice.recordTranscript called with role 'user' and transcript content | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'user transcript'` |
| 2 | An active voice session | Assistant responds and response.audio_transcript.done event received | voice.recordTranscript called with role 'agent' and transcript content | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'agent transcript'` |
| 3 | Transcript recording fails (Convex error) | Error occurs during recordTranscript | Error logged but voice session continues uninterrupted | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'transcript error'` |
| 4 | Multiple rapid transcript events | Events arrive in quick succession | All transcripts recorded in correct chronological order | `pnpm vitest run --reporter=verbose 2>&1 \| grep 'transcript order'` |
