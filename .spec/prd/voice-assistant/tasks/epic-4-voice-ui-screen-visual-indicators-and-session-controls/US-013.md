## CRITICAL CONSTRAINTS

### MUST
- Show mic icon when IDLE (tappable to start), stop icon when active (tappable to stop)
- Include testID: 'voice-mic-button'
- Handle press with debounce to prevent rapid start/stop (H-VSESS-01 scenario)

### NEVER
- Allow starting a new session while CONNECTING is in progress
- Use hardcoded colors or sizes
- Render without accessibility label

### STRICTLY
- Use Pressable with interactive states (pressed, disabled) per CLAUDE.md pattern
- Co-locate a .stories.tsx file

## SPECIFICATION

**Objective:** Create the mic/stop toggle button that starts and stops voice sessions

**Success looks like:** Button toggles between mic and stop icons based on voice state, handles rapid taps gracefully, and follows all design rules

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | Voice state is IDLE | VoiceMicButton renders | Shows mic icon, tappable, onStart called when pressed | `pnpm vitest run --reporter=verbose 2>&1 \| grep VoiceMicButton` |

### Interaction Notes

- Disabled state during CONNECTING prevents double-start

### Code Pattern
Source: CLAUDE.md (Pressable interactive states pattern)

```typescript
<Pressable
  onPress={onPress}
  disabled={disabled}
  testID="voice-mic-button"
  accessibilityLabel={isActive ? 'Stop voice session' : 'Start voice session'}
>
  {({ pressed }) => (
    <View style={{
      backgroundColor: pressed
        ? semantic.color.primary.pressed
        : disabled
        ? semantic.color.primary.disabled
        : semantic.color.primary.default,
      borderRadius: semantic.radius.full,
      padding: semantic.space.md,
    }}>
      <Icon name={isActive ? 'stop' : 'microphone'} />
    </View>
  )}
</Pressable>
```

### Anti-pattern (DO NOT)
Do NOT use TouchableOpacity. Use Pressable with state-driven styles.
