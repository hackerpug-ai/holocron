# US-X-003: Screen Reader Accessibility

> Task ID: US-X-003
> Type: FEATURE
> Priority: P0
> Estimate: 90 minutes
> Assignee: frontend-designer

## CRITICAL CONSTRAINTS

### MUST
- All cards pass accessibility audit
- Proper accessibility labels on all interactive elements
- Screen reader support (VoiceOver/TalkBack)
- High contrast in both light/dark modes
- Minimum touch target size (44x44)

### NEVER
- Skip accessibility labels on buttons
- Use color alone to convey information
- Create traps in navigation
- Block screen reader navigation

### STRICTLY
- Use semantic accessibility props
- Test with VoiceOver on iOS
- Follow WCAG 2.1 AA guidelines

## SPECIFICATION

**Objective:** Ensure all feed components are fully accessible to screen reader users

**Success looks like:** User with VoiceOver can navigate feed, understand card content, provide feedback, and open articles without visual cues

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | Screen reader active | User focuses card | Card type, title, summary, source announced | VoiceOver verification |
| 2 | User focuses feedback button | Screen reader announces | "More like this, button, not pressed" | Accessibility inspector |
| 3 | User activates feedback button | Action fires | "Liked" or "Unliked" announced | VoiceOver verification |
| 4 | User navigates feed | Swipe gestures | Cards traversed in logical order | VoiceOver verification |
| 5 | High contrast mode | Cards render | All text readable, icons visible | Visual verification |
| 6 | User focuses filter chip | Screen reader announces | "Video, 12 items, button, not selected" | Accessibility inspector |

### Accessibility Pattern

```typescript
// Card component with full accessibility
export function VideoCard({ title, summary, source, duration, onPress }: Props) {
  return (
    <Pressable
      onPress={onPress}
      accessible={true}
      accessibilityRole="button"
      accessibilityLabel={`${title}. ${summary}. ${source}. ${duration}. Video. Tap to watch.`}
      accessibilityHint="Opens video in web view"
      testID="video-card"
    >
      {/* Visual content */}
      <FastImage ... />
      <Text>{title}</Text>
      <Text>{summary}</Text>
      
      {/* Feedback with accessibility */}
      <FeedbackButtons
        findingId={url}
        currentFeedback={feedback}
        onFeedback={handleFeedback}
      />
    </Pressable>
  )
}

// Feedback buttons with accessibility
export function FeedbackButtons({ currentFeedback, onFeedback }: Props) {
  return (
    <View 
      accessible={true}
      accessibilityRole="none"
      accessibilityLabel="Feedback options"
    >
      <Pressable
        onPress={() => onFeedback('positive')}
        accessible={true}
        accessibilityRole="button"
        accessibilityLabel="More like this"
        accessibilityHint="Tap to indicate you want more content like this"
        accessibilityState={{ 
          selected: currentFeedback === 'positive',
          expanded: false 
        }}
        testID="feedback-thumbs-up"
      >
        <ThumbsUp 
          size={18}
          fill={currentFeedback === 'positive' ? 'currentColor' : 'none'}
        />
      </Pressable>
      
      <Pressable
        onPress={() => onFeedback('negative')}
        accessible={true}
        accessibilityRole="button"
        accessibilityLabel="Less like this"
        accessibilityHint="Tap to indicate you want less content like this"
        accessibilityState={{ selected: currentFeedback === 'negative' }}
        testID="feedback-thumbs-down"
      >
        <ThumbsDown 
          size={18}
          fill={currentFeedback === 'negative' ? 'currentColor' : 'none'}
        />
      </Pressable>
    </View>
  )
}

// Filter chips with accessibility
export function FeedFilterChips({ categories, selectedCategory, onCategoryChange }: Props) {
  return (
    <ScrollView 
      horizontal
      accessible={true}
      accessibilityRole="tablist"
      accessibilityLabel="Filter by category"
    >
      {CATEGORIES.map((cat) => (
        <Pressable
          key={cat}
          onPress={() => onCategoryChange(cat)}
          accessible={true}
          accessibilityRole="button"
          accessibilityState={{ selected: selectedCategory === cat }}
          accessibilityLabel={`${cat}. ${categories[cat]} items`}
          testID={`filter-chip-${cat}`}
        >
          <Text>{cat} ({categories[cat]})</Text>
        </Pressable>
      ))}
    </ScrollView>
  )
}

// Feed list with accessibility
export function SubscriptionFeedScreen() {
  return (
    <View accessible={true} accessibilityLabel="What's New feed">
      <FlatList
        data={findings}
        renderItem={({ item, index }) => (
          <FeedCard 
            item={item}
            accessible={true}
            accessibilityLabel={`Item ${index + 1} of ${findings.length}`}
          />
        )}
        accessibilityRole="list"
        accessibilityLabel="Content feed"
      />
    </View>
  )
}
```

### Accessibility Checklist

```
For each interactive element:
- [ ] accessibilityRole (button, link, image, etc.)
- [ ] accessibilityLabel (what it is)
- [ ] accessibilityHint (what it does, optional)
- [ ] accessibilityState (selected, disabled, etc.)
- [ ] accessible={true} (or group with accessibilityRole="none")

For images:
- [ ] accessibilityLabel (describe content)
- [ ] accessibilityRole="image"

For lists:
- [ ] accessibilityRole="list"
- [ ] Items have accessibilityRole="listitem"

For groups:
- [ ] accessibilityRole="none" with nested accessible elements
- [ ] Or accessibilityRole with group label
```

### Anti-pattern (DO NOT)
```typescript
// ❌ WRONG: No accessibility props
<Pressable onPress={onPress}>
  <ThumbsUp />
</Pressable>

// ❌ WRONG: Generic label
accessibilityLabel="Button" // What button?

// ❌ WRONG: Color-only indication
{isSelected && <View className="bg-primary" />} // Screen reader can't see color
```
