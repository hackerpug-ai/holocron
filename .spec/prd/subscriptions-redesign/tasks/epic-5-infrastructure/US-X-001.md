# US-X-001: Performance with Rich Media

> Task ID: US-X-001
> Type: FEATURE
> Priority: P0
> Estimate: 120 minutes
> Assignee: frontend-designer

## CRITICAL CONSTRAINTS

### MUST
- Feed loads in under 2 seconds on 4G connection
- Images load progressively (low-res → high-res)
- Cards render skeleton states while loading
- Scroll performance is smooth (60fps)
- Memory usage stays under 200MB

### NEVER
- Block feed rendering waiting for all images
- Cause frame drops during scroll
- Load full-resolution images for offscreen cards
- Memory leak from unoptimized image caching

### STRICTLY
- Use FlatList with performance optimizations
- Use react-native-fast-image for caching
- Implement proper list virtualization

## SPECIFICATION

**Objective:** Ensure the feed performs well with rich media content (images, thumbnails)

**Success looks like:** User scrolls feed smoothly at 60fps, images load progressively, feed renders in under 2 seconds

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | User opens feed | Screen loads | Feed visible in under 2 seconds | Performance profiling |
| 2 | User scrolls feed | Scroll gesture | 60fps maintained, no frame drops | Flipper profiling |
| 3 | Images loading | Card renders | Skeleton shows, then image fades in | Visual verification |

### FlatList Optimization Pattern

```typescript
// components/subscriptions/SubscriptionFeedScreen.tsx
import { FlatList, View } from 'react-native'

export function SubscriptionFeedScreen() {
  return (
    <FlatList
      data={findings}
      keyExtractor={(item) => item.url}
      renderItem={({ item }) => <FeedCard item={item} />}
      
      // Performance optimizations
      removeClippedSubviews={true}        // Unmount offscreen views
      maxToRenderPerBatch={10}            // Limit render batch
      updateCellsBatchingPeriod={50}      // Batch updates
      initialNumToRender={10}             // Initial render count
      windowSize={5}                      // Render window (5 screens)
      getItemType={(item) => item.category} // Stable type for recycling
      
      // List optimizations
      showsVerticalScrollIndicator={false}
      keyboardDismissMode="on-drag"
      keyboardShouldPersistTaps="handled"
      
      // Pull to refresh
      refreshControl={
        <RefreshControl
          refreshing={isRefreshing}
          onRefresh={handleRefresh}
        />
      }
      
      // Skeleton while loading
      ListEmptyComponent={
        isLoading ? <FeedSkeleton count={5} /> : <EmptyState />
      }
    />
  )
}

// components/subscriptions/FeedSkeleton.tsx
export function FeedSkeleton({ count = 5 }: { count?: number }) {
  return (
    <View className="gap-4 p-4">
      {Array.from({ length: count }).map((_, i) => (
        <View 
          key={i} 
          className="rounded-2xl border border-border bg-card overflow-hidden"
          testID={`feed-card-skeleton-${i}`}
        >
          {/* Image skeleton */}
          <Skeleton className="h-40 w-full" />
          
          {/* Content skeleton */}
          <View className="p-4 gap-2">
            <Skeleton className="h-4 w-3/4" />
            <Skeleton className="h-3 w-1/2" />
          </View>
        </View>
      ))}
    </View>
  )
}
```

### Image Optimization Pattern

```typescript
// Use FastImage with proper caching
import FastImage from 'react-native-fast-image'

// Preload critical images
FastImage.preload([
  { uri: thumbnailUrl1 },
  { uri: thumbnailUrl2 },
])

// In card component
<FastImage
  source={{ 
    uri: thumbnailUrl,
    priority: FastImage.priority.normal,
    cache: FastImage.cacheControl.web
  }}
  style={styles.thumbnail}
  resizeMode={FastImage.resizeMode.cover}
  // Show fallback while loading
  defaultSource={require('@/assets/images/placeholder.png')}
/>
```

### Anti-pattern (DO NOT)
```typescript
// ❌ WRONG: No virtualization
<ScrollView>
  {findings.map(f => <FeedCard item={f} />)}

// ❌ WRONG: Loading all images at once

// ❌ WRONG: No skeleton

```
