EXAMPLES
--------------------------------------------------------------------------------

**Follow this exact pattern (public action wrapper):**

```typescript
// Location: convex/feeds/actions.ts

import { action } from "../_generated/server";
import { internal } from "../_generated/api";

// ============================================================================
// Feed Building Actions
// ============================================================================

/**
 * Build feed from recent subscription content (public wrapper)
 *
 * This is a public wrapper around internal.feeds.internal.buildFeed
 * that includes authentication checks.
 *
 * Use this for:
 * - Manual testing via dashboard
 * - Admin-triggered feed refresh
 * - On-demand feed building
 *
 * @returns Object with processed count and feed items created
 */
export const buildFeed = action({
  args: {},
  handler: async (ctx) => {
    // Authentication check
    const user = await ctx.auth.getUserIdentity();
    if (!user) {
      throw new Error("Unauthorized: User must be logged in to build feed");
    }

    // Call internal action to do the actual work
    const result = await ctx.runAction(internal.feeds.internal.buildFeed, {});

    return result;
  },
});
```

**Why this pattern:**
- Public action provides authenticated access to internal function
- No duplicate logic - delegates to internal action
- Authentication check prevents unauthorized access
- Same return type for consistency
- Easy to test via Convex dashboard

**Anti-pattern (DO NOT do this):**

```typescript
// DON'T: Duplicate feed building logic
export const buildFeed = action({
  args: {},
  handler: async (ctx) => {
    // All the grouping/creation logic copied from internal.ts
    // This violates DRY and creates maintenance burden
    const content = await ctx.runQuery(/* ... */);
    const grouped = /* ... */;
    // ...
  },
});

// DON'T: Skip authentication
export const buildFeed = action({
  args: {},
  handler: async (ctx) => {
    // No auth check = anyone can trigger feed builds!
    return await ctx.runAction(internal.feeds.internal.buildFeed, {});
  },
});
```
