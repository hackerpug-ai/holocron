EXAMPLES
--------------------------------------------------------------------------------

**Follow this exact pattern (cron job with interval):**

```typescript
// Location: convex/crons.ts (add after existing cron jobs)

/**
 * Feed Builder
 *
 * Builds feed items from recent subscription content.
 * - Runs every 2 hours
 * - Groups content by creator
 * - Creates feed items and marks content as in feed
 *
 * This ensures the feed is regularly updated with new content
 * from all active subscriptions.
 *
 * @see .spec/prd/subscription-feed-redesign/tasks/02-feed-building/FR-009.md
 */
crons.interval(
  "feed-builder",
  { hours: 2 }, // Run every 2 hours
  internal.feeds.internal.buildFeed,
  { timeoutMinutes: 10 } // Maximum execution time
);
```

**Why this pattern:**
- Descriptive comment explains purpose and schedule
- Uses interval (not specific time) for consistent updates
- Calls internal action directly (no authentication needed)
- Timeout prevents runaway jobs
- Reference to task file for maintenance

**Anti-pattern (DO NOT do this):**

```typescript
// DON'T: Use public action (requires auth)
crons.interval(
  "feed-builder",
  { hours: 2 },
  api.feeds.actions.buildFeed,  // Wrong - needs auth!
  { timeoutMinutes: 10 }
);

// DON'T: Run too frequently
crons.interval(
  "feed-builder",
  { minutes: 15 },  // Wrong - too frequent, wastes resources
  internal.feeds.internal.buildFeed
);

// DON'T: Skip timeout
crons.interval(
  "feed-builder",
  { hours: 2 },
  internal.feeds.internal.buildFeed
  // Missing timeout = no limit on execution time
);
```
