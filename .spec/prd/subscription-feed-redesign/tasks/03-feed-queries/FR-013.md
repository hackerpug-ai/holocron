**Follow this exact pattern (add to queries.ts after getByCreator):**

```typescript
// Location: convex/feeds/queries.ts (add after getByCreator)

export const getUnviewedCount = query({
  args: {
    creatorProfileId: v.optional(v.id("creatorProfiles")),
  },
  handler: async (ctx, args) => {
    // Use by_viewed index for counting unviewed items
    if (args.creatorProfileId) {
      // Count unviewed for specific creator
      const items = await ctx.db
        .query("feedItems")
        .withIndex("by_viewed", (q) =>
          q.eq("viewed", false)
        )
        .filter((q) => q.eq("creatorProfileId", args.creatorProfileId))
        .collect();

      return items.length;
    }

    // Count all unviewed items
    const items = await ctx.db
      .query("feedItems")
      .withIndex("by_viewed", (q) =>
        q.eq("viewed", false)
      )
      .collect();

    return items.length;
  },
});
```

**Why this pattern:**
- Uses by_viewed index for optimal performance
- Returns count only (not items) for fast badge updates
- Supports optional creator filtering (creator-specific badges)
- Uses .collect() then .length (Convex doesn't have count aggregation yet)
- Filters for viewed=false explicitly

**Anti-pattern (DO NOT do this):**

```typescript
// DON'T: Return items instead of count
export const getUnviewedCount = query({
  handler: async (ctx) => {
    // BAD: Returns all items, client counts them
    return await ctx.db.query("feedItems").collect();
    // Client: items.filter(i => !i.viewed).length
    // Wasteful: transferring all items just to count
  },
});

// DON'T: Query without index
export const getUnviewedCount = query({
  handler: async (ctx) => {
    // BAD: Full table scan
    const all = await ctx.db.query("feedItems").collect();
    return all.filter(item => !item.viewed).length;
    // Slow on large datasets
  },
});

// DON'T: Hardcode viewed filter incorrectly
export const getUnviewedCount = query({
  handler: async (ctx) => {
    // BAD: Using undefined instead of false
    const items = await ctx.db
      .query("feedItems")
      .withIndex("by_viewed", (q) => q.eq("viewed", undefined))
      .collect();
    return items.length;
  },
});
```

**Note on future optimization:**
When Convex adds native count aggregation, this query should be updated to use it. For now, .collect().length is the correct pattern.
