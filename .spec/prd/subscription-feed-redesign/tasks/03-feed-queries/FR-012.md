EXAMPLES

**Follow this exact pattern (add to queries.ts after getFeed):**

```typescript
// Location: convex/feeds/queries.ts (add after getFeed)

export const getByCreator = query({
  args: {
    creatorProfileId: v.id("creatorProfiles"),
    limit: v.optional(v.number()),
  },
  handler: async (ctx, args) => {
    const limit = args.limit ?? 20;

    // Use by_creator index for optimal performance
    const items = await ctx.db
      .query("feedItems")
      .withIndex("by_creator", (q) =>
        q.eq("creatorProfileId", args.creatorProfileId)
      )
      .order("desc")
      .take(limit);

    return items;
  },
});
```

**Why this pattern:**
- Required creatorProfileId prevents accidental unfiltered queries
- Uses dedicated by_creator index for fast lookups
- Default limit of 20 (creator pages show fewer items)
- Ordered by discoveredAt desc (newest content first)
- Simple, focused query (single purpose)
