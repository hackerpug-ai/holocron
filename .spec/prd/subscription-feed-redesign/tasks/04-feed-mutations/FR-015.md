EXAMPLES
--------------------------------------------------------------------------------

**Follow this exact pattern for mutations:**

```typescript
// Location: convex/feeds/mutations.ts

import { mutation } from "./_generated/server";
import { v } from "convex/values";
import { markViewedArgs } from "./validators";

export const markViewed = mutation({
  args: markViewedArgs,
  handler: async (ctx, args) => {
    const { feedItemIds, viewedAt = Date.now() } = args;

    // Update all specified items
    await Promise.all(
      feedItemIds.map((id) =>
        ctx.db.patch(id, {
          viewed: true,
          viewedAt,
        })
      )
    );

    return { marked: feedItemIds.length };
  },
});
```

**Why this pattern:**
- Destructure args with default for viewedAt
- Use Promise.all() for concurrent updates
- Return count of marked items
- Export named function for imports
- Use validator from validators.ts

**Anti-pattern (DO NOT do this):**

```typescript
// DON'T: Update viewed without viewedAt
export const markViewed = mutation({
  args: markViewedArgs,
  handler: async (ctx, args) => {
    await Promise.all(
      args.feedItemIds.map((id) =>
        ctx.db.patch(id, { viewed: true })  // Missing viewedAt!
      )
    );
  },
});

// DON'T: Loop sequentially (slow)
for (const id of feedItemIds) {
  await ctx.db.patch(id, { viewed: true, viewedAt });
}

// DON'T: Ignore validation
export const markViewed = mutation({
  args: {  // Should import from validators.ts
    feedItemIds: v.array(v.id("feedItems")),
  },
  // ...
});
```
