BACKGROUND
--------------------------------------------------------------------------------

**Problem:** Two competing SubscriptionFeedScreen implementations exist:
1. `components/subscriptions/SubscriptionFeedScreen.tsx` — FlatList-based, uses hook, but NO WebView, NO filters, NO card variants, stub refresh/loadMore
2. `screens/subscriptions-feed-screen.tsx` — ScrollView-based, inline Convex queries, HAS WebView+filters+settings, but NO FlatList, NO infinite scroll

The route `app/subscriptions/index.tsx` imports the components/ version (the stub).

**Why it matters:** All EPIC-06 card variants, EPIC-07 filters, and EPIC-08 WebView integration are dead code in the live app.

**Current state:** Two files, route uses the less-complete one.

**Desired state:** One file, route uses the fully-featured one.
