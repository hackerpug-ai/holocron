EXAMPLES
--------------------------------------------------------------------------------

**Follow this exact pattern from convex/schema.ts:246-269:**

```typescript
subscriptionContent: defineTable({
  sourceId: v.id("subscriptionSources"),
  contentId: v.string(),
  title: v.string(),
  url: v.optional(v.string()),
  metadataJson: v.optional(v.any()),
  passedFilter: v.boolean(),
  filterReason: v.optional(v.string()),
  researchStatus: v.union(
    v.literal("pending"),
    v.literal("queued"),
    v.literal("researched"),
    v.literal("skipped")
  ),
  discoveredAt: v.number(),
  researchedAt: v.optional(v.number()),
  documentId: v.optional(v.id("documents")),
  embedding: v.optional(v.array(v.float64())),
})
  .index("by_source", ["sourceId", "discoveredAt"])
  .index("by_source_content", ["sourceId", "contentId"])
  .index("by_status", ["researchStatus"])
  .index("by_content_id", ["contentId"])
  .vectorIndex("by_embedding", {
    vectorField: "embedding",
    dimensions: 1024,
  }),
```

**Why this pattern:**
- Uses defineTable with v.id() for foreign keys
- Optional fields use v.optional()
- Union literals for fixed sets of values
- Indexes defined chained after table
- Vector index follows same pattern as documents table

**Anti-pattern (DO NOT do this):**

```typescript
// DON'T: Use v.null() instead of v.optional()
url: v.null(v.string()),

// DON'T: Forget commas between fields
title: v.string()
url: v.optional(v.string()),

// DON'T: Use wrong union syntax
contentType: v.union("video", "blog", "social"),
```
