EXAMPLES

**Follow this exact pattern from convex/schema.ts:42-49 (documents table optional fields):**

```typescript
documents: defineTable({
  title: v.string(),
  content: v.string(),
  category: v.string(),
  filePath: v.optional(v.string()),
  fileType: v.optional(v.string()),
  status: v.optional(v.string()),
  date: v.optional(v.string()),
  time: v.optional(v.string()),
  // ... more fields
})
```

**Why this pattern:**
- Required fields first (v.string(), v.number(), v.boolean())
- Optional fields after (v.optional(...))
- Optional foreign keys use v.optional(v.id("tableName"))
- Optional arrays use v.optional(v.array(...))

**Anti-pattern (DO NOT do this):**

```typescript
// DON'T: Mix required and optional fields randomly
url: v.optional(v.string()),
title: v.string(),  // Required after optional = confusing

// DON'T: Use v.null()
feedItemId: v.null(v.id("feedItems")),

// DON'T: Forget v.optional() for new fields to existing table
thumbnailUrl: v.string(),  // BREAKS existing data
```
