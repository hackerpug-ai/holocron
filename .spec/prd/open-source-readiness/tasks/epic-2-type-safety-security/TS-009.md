# TS-009: Fix `process.env` Merge in MCP Config

> Task ID: TS-009
> Type: SECURITY
> Priority: P1
> Estimate: 15 minutes
> Assignee: backend-implementer

## CRITICAL CONSTRAINTS

### MUST
- Replace `process.env = { ...process.env, ...Bun.env }` spread in MCP config
- Use targeted key merging for specific environment variables
- Document which keys are merged

### NEVER
- Spread entire `Bun.env` into `process.env` (security risk)
- Merge all environment variables indiscriminately

### STRICTLY
- Only merge known, required keys

## SPECIFICATION

**Objective:** Fix unsafe environment variable merging in the MCP server config.

**Success looks like:** Only specific, required environment variables are merged from `Bun.env` to `process.env`.

## GUARDRAILS

### WRITE-ALLOWED
- `holocron-mcp/src/config/env.ts` (MODIFY) - Replace spread merge with targeted merging

### WRITE-PROHIBITED
- Spreading entire `Bun.env` or `process.env`
- Merging unknown/unlisted environment variables

## DESIGN

### References
- PRD Section 2.4, AC-25

### Code Pattern
```typescript
// Before — DANGEROUS (merges ALL env vars)
process.env = {
  ...process.env,
  ...Bun.env,
};

// After — Targeted, safe
const KEYS_TO_MERGE = [
  "OPENAI_API_KEY",
  "ANTHROPIC_API_KEY",
  "CONVEX_DEPLOYMENT",
  "CONVEX_ADMIN_KEY",
] as const;

for (const key of KEYS_TO_MERGE) {
  if (Bun.env[key]) {
    process.env[key] = Bun.env[key] as string;
  }
}

// Or even more explicit
if (Bun.env.OPENAI_API_KEY) {
  process.env.OPENAI_API_KEY = Bun.env.OPENAI_API_KEY as string;
}
if (Bun.env.ANTHROPIC_API_KEY) {
  process.env.ANTHROPIC_API_KEY = Bun.env.ANTHROPIC_API_KEY as string;
}
```

## CODING STANDARDS

- **security**: Never spread entire environment objects
- **explicit**: Always list specific keys being merged

## DEPENDENCIES

No dependencies.

## NOTES

- Spreading `Bun.env` can inadvertently expose or override sensitive variables
- List only the keys that are actually needed by the MCP server
- Document in comments why each key is needed
- Consider using a typed configuration object instead
- Test that the MCP server still has access to required keys after the change
