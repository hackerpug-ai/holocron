# TS-007: Guard or Remove `clearAll` Mutations

> Task ID: TS-007
> Type: SECURITY
> Priority: P1
> Estimate: 30 minutes
> Assignee: convex-implementer

## CRITICAL CONSTRAINTS

### MUST
- Find all `clearAll` mutations across the codebase
- Either remove them entirely OR guard with admin check
- At minimum: use environment variable guard (`process.env.ALLOW_CLEAR_ALL`)
- Document the guard in comments

### NEVER
- Leave unguarded `clearAll` mutations in the codebase
- Use client-side-only checks (can be bypassed)

### STRICTLY
- All destructive mutations must be server-gated

## SPECIFICATION

**Objective:** Remove or guard all `clearAll` mutations to prevent unauthorized data deletion.

**Success looks like:** No unguarded `clearAll` mutations exist; all destructive operations have admin guards.

## GUARDRAILS

### WRITE-ALLOWED
- Convex mutation files (MODIFY) - Add guards to `clearAll` mutations
- `.env.example` (MODIFY) - Document the guard variable

### WRITE-PROHIBITED
- Using client-side-only guards (user can bypass)
- Removing guards without removing the mutation

## DESIGN

### References
- PRD Section 2.4, AC-23

### Code Pattern
```typescript
// Before — DANGEROUS
export const clearAllData = mutation({
  handler: async (ctx) => {
    const messages = await ctx.db.query("chatMessages").collect();
    for (const message of messages) {
      await ctx.db.delete(message._id);
    }
  }
});

// After — Guarded
export const clearAllData = mutation({
  handler: async (ctx) => {
    // SECURITY: Only allow in development/admin mode
    if (process.env.ALLOW_CLEAR_ALL !== "true") {
      throw new Error("clearAll is disabled in production");
    }

    const messages = await ctx.db.query("chatMessages").collect();
    for (const message of messages) {
      await ctx.db.delete(message._id);
    }
  }
});

// Alternative: Remove entirely and delete the export
```
