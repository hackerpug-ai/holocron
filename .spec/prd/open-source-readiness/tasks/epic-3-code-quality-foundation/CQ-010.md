### WRITE-ALLOWED
- Dead doc files (DELETE via git rm) - Remove tracked files
- Proper documentation (MODIFY) - Integrate content if valuable

### WRITE-PROHIBITED
- Deleting files without reading their content first
- Removing files that contain useful information

## DESIGN

### References

1. Read each file to understand its content
2. Determine if content is valuable:

   - If no → delete with `git rm`
3. Commit the removal with descriptive message

## CODING STANDARDS

- **documentation**: Keep only living, useful documentation
