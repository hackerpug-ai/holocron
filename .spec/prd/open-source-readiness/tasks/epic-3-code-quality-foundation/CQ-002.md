# CQ-002: Delete or Replace `toBeDefined()` Tests

> Task ID: CQ-002
> Type: REFACTOR
> Priority: P1
> Estimate: 45 minutes
> Assignee: red-test-generator

## CRITICAL CONSTRAINTS

### MUST
- Find all tests that only check `expect(fn).toBeDefined()`
- Either delete them (if they provide zero value) or replace with behavioral tests
- Document which tests were deleted vs replaced

### NEVER
- Keep tests that only check existence — they provide no regression protection

### STRICTLY
- All remaining tests must verify actual behavior

## SPECIFICATION

**Objective:** Remove all existence-check tests that don't verify actual behavior.

**Success looks like:** Zero tests that only check `toBeDefined()`. All remaining tests verify behavior.

## DESIGN

### References

### Code Pattern
```typescript
// BEFORE — Delete this (provides no value)
describe('MyComponent', () => {
  it('exists', () => {
    expect(MyComponent).toBeDefined();
  });
});

// AFTER — Replace with behavioral test
describe('MyComponent', () => {
  it('renders title prop correctly', () => {
    const { getByText } = render(<MyComponent title="Test" />);
    expect(getByText('Test')).toBeTruthy();
  });

  it('calls onPress when clicked', () => {
    const onPress = jest.fn();
    const { getByTestId } = render(<MyComponent onPress={onPress} />);
    fireEvent.press(getByTestId('my-component-button'));
    expect(onPress).toHaveBeenCalledTimes(1);
  });
});
```

## CODING STANDARDS

- **testing**: Tests must verify behavior, not existence
- **quality**: If a test provides no regression protection, delete it
