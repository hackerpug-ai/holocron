### Bug Report Template
```markdown
---
name: Bug report
about: Report a problem with Holocron
title: '[BUG] '
labels: bug
assignees: ''
---

**Describe the bug**
A clear description of what the bug is.

**To Reproduce**
Steps to reproduce the behavior:
1. Go to '...'
2. Click on '....'
3. Scroll down to '....'
4. See error

**Expected behavior**
What you expected to happen.

**Screenshots**
If applicable, add screenshots to help explain your problem.

**Environment**
- Device: [e.g. iPhone 14, Pixel 6]
- OS: [e.g. iOS 17, Android 14]
- App Version: [e.g. 1.0.0]
- Node version: [e.g. 18.0.0]

**Additional context**
Add any other context about the problem here.

**Convex Deployment**
- Deployment URL: (if applicable)
- Any Convex errors in logs?
```

### Feature Request Template
```markdown
---
name: Feature request
about: Suggest an idea for Holocron
title: '[FEATURE] '
labels: enhancement
---

**Is your feature request related to a problem?**
A clear description of what the problem is.

**Describe the solution you'd like**
A clear description of what you want to happen.

**Describe alternatives you've considered**
A clear description of any alternative solutions or features you've considered.

**Additional context**
Add any other context or screenshots about the feature request here.
```

### PR Template
```markdown
## Description
Briefly describe what this PR changes and why.

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation update

## Testing
- [ ] Tests pass locally
- [ ] New tests added (if applicable)
- [ ] Manual testing completed

## Checklist
- [ ] Code follows project style guidelines
- [ ] No new type errors (`pnpm tsc --noEmit`)
- [ ] No new lint errors
- [ ] Documentation updated (if needed)
- [ ] Commit messages follow conventional commits

## Related Issues
Fixes #
Related to #
```

### CoC (Contributor Covenant)
```markdown
# Contributor Covenant Code of Conduct

## Our Pledge
... (use standard Contributor Covenant text)
...

## Contact
[Email or contact method for moderation]
```

## CODING STANDARDS

- **templates**: Make them easy to fill out
- **complete**: Request all necessary info
- **friendly**: Keep tone welcoming

No dependencies.
