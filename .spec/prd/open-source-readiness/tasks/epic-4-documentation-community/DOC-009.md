### CONTRIBUTING.md Template
```markdown
# Contributing to Holocron

Thanks for your interest in contributing! This document covers how to get started.

## Getting Started

### Prerequisites
- Node.js 18+
- pnpm
- iOS Simulator (Mac) or Android Emulator

### Setup

\`\`\`bash
# Fork and clone the repo
git clone https://github.com/yourusername/holocron.git
cd holocron

# Install dependencies
pnpm install

# Copy environment template
cp .env.example .env
# Add your API keys

# Start development
pnpm dev  # Convex backend
pnpm start  # Expo dev server (in new terminal)
\`\`\`

## Development Workflow

### Branch Strategy
- `main` — Protected, only merge commits
- `feature/` — New features
- `fix/` — Bug fixes
- `refactor/` — Code refactoring

### Making Changes

1. Create a branch from `main`:
   \`\`\`bash
   git checkout -b feature/your-feature-name
   \`\`\`

2. Make your changes following our [code style](#code-style)

3. Test your changes:
   \`\`\`bash
   pnpm tsc --noEmit  # Type check
   pnpm vitest run    # Run tests
   pnpm lint          # Lint code
   \`\`\`

4. Commit with conventional commits:
   \`\`\`bash
   git commit -m "feat: add new research feature"
   \`\`\`

## Pull Requests

1. Push your branch:
   \`\`\`bash
   git push origin feature/your-feature-name
   \`\`\`

2. Open a PR from your fork/branch to `main`

3. Ensure:
   - All tests pass
   - No type errors
   - PR description follows the template
   - At least one approval is required

### PR Template

See `.github/PULL_REQUEST_TEMPLATE.md` for the PR template.

## Code Style

### TypeScript
- Use explicit types (no `any`)
- Follow React/React Native patterns
- Use `Id<"tableName">` for Convex IDs

### React Native
- Use `react-native-paper` Text component
- Follow theme tokens (no hardcoded colors)
- Add `testID` to interactive elements

### Convex
- Use `ActionCtx`, `MutationCtx`, `QueryCtx` explicitly
- Prefer `v.union(v.literal(...))` over `v.string()`
- Avoid `v.any()` — use structured validators

### Testing
- Write behavioral tests, not `toBeDefined()` checks
- Test edge cases and boundaries
- Use `convex-test` for backend tests

## Getting Help

- Open an issue for bugs or feature requests
- Start discussions for questions
- Check existing issues and PRs first

Thanks for contributing! 🎉
```
