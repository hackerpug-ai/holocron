### Verification Checklist

1. **File Extensions**
   - [ ] Web-specific files use `.web.` extension
   - [ ] Native-specific files use `.native.` extension
   - [ ] Shared files have no extension

2. **Import Patterns**
   - [ ] Platform modules imported without extension
   - [ ] Metro/React Native resolves correct file

3. **No Web Imports in Mobile**
   - [ ] No `react-router-dom` in mobile code
   - [ ] No `react-dom` in mobile code
   - [ ] No web-only APIs in mobile

### Correct Pattern
```typescript
// ✅ GOOD: Platform-specific files
// app/navigation.web.tsx - Web navigation
// app/navigation.native.tsx - Mobile navigation
// app/navigation.ts - Shared (if any)

// Import without extension - React Native resolves:
import { Navigation } from './navigation'

// ✅ GOOD: Conditional import in shared code
import { Platform } from 'react-native'

let Component: React.ComponentType
if (Platform.OS === 'web') {
  Component = require('./Component.web').default
} else {
  Component = require('./Component.native').default
}

// ❌ BAD: Importing web-only module in mobile
// In a .native.tsx file:
import { BrowserRouter } from 'react-router-dom' // ❌ Wrong!
```

### Finding Issues
```bash
# List platform-specific files
find app/ components/ -name "*.web.*" -o -name "*.native.*"

# Find web imports in non-web files
grep -r "react-router-dom" app/ | grep -v "\.web\."

# Find react-dom usage
grep -r "from ['\"]react-dom['\"]" app/ | grep -v "\.web\."
```

## CODING STANDARDS

- **platform**: Use file extensions for platform separation
- **imports**: Let Metro resolve platform files

## DEPENDENCIES

No dependencies.

## NOTES

- React Native Metro bundler resolves .web. / .native. automatically
- Web-only modules (react-router-dom, react-dom) break mobile builds
- Keep platform separation clean
- Shared code should work on both platforms
