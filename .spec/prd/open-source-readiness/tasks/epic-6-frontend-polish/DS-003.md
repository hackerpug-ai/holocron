# DS-003: Replace Hardcoded Typography with Theme Tokens

> Task ID: DS-003
> Type: REFACTOR
> Priority: P1
> Estimate: 30 minutes
> Assignee: react-native-implementer

## CRITICAL CONSTRAINTS

### MUST
- Replace all hardcoded typography with `semantic.type.*` tokens
- Prefer react-native-paper Text variants (`variant="titleLarge"`)
- Only use semantic.type when Paper variant unavailable

### NEVER
- Hardcode `fontSize`, `fontWeight`, `lineHeight`

### STRICTLY
- All typography uses theme tokens or Paper variants

## SPECIFICATION

**Objective:** Replace all hardcoded typography with theme tokens or Paper variants.

**Success looks like:** Zero hardcoded typography values in component files.

```bash

# Find fontWeight
```

### Replacement Pattern
```typescript

  Title
</Text>

<Text style={{ fontSize: semantic.type.heading.lg.fontSize }}>
  Custom Text
</Text>
```

## CODING STANDARDS

- **typography**: Use Paper variants first
- **fallback**: Use semantic.type only when Paper doesn't suffice
- **import**: Always use `Text` from react-native-paper

## DEPENDENCIES

No dependencies.

## NOTES

- Paper variants cover 95% of use cases
- Only use semantic.type for custom font sizes
- Check that you're importing Text from react-native-paper
- This improves accessibility and consistency
- Paper automatically handles dark mode contrast
