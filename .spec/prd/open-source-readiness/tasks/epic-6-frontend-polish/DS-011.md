### Verification Checklist

1. **Web Components**
   - [ ] Inputs have associated `<label>` elements
   - [ ] Labels use `htmlFor` to link to inputs
   - [ ] `aria-label` used where labels aren't visible

2. **React Native Components**
   - [ ] TextInputs have visible labels
   - [ ] `accessibilityLabel` prop used
   - [ ] Labels are descriptive

### Correct Pattern
```typescript
// ✅ GOOD: Web with label
<label htmlFor="email-input">
  Email
</label>
<input
  id="email-input"
  data-testid="login-screen-email-input"
  value={email}
  onChange={(e) => setEmail(e.target.value)}
/>

// ✅ GOOD: React Native with label
<Text accessibilityLabel="Email">Email</Text>
<TextInput
  accessibilityLabel="Email input field"
  testID="login-screen-email-input"
  value={email}
  onChangeText={setEmail}
/>

// ✅ GOOD: aria-label for icon-only buttons
<button
  aria-label="Close dialog"
  data-testid="dialog-close-button"
  onClick={onClose}
>
  <XIcon />
</button>

// ❌ BAD: Unlabeled input
<input
  value={email}
  onChange={(e) => setEmail(e.target.value)}
/>
```

### Finding Issues
```bash
# Find inputs without labels (web)
grep -A3 -B3 "<input" components/web/ | grep -v "label\|aria-label"

# Find TextInputs without labels (RN)
grep -A3 -B3 "TextInput" app/ components/ | grep -v "Label\|accessibilityLabel"
```

## CODING STANDARDS

- **accessibility**: All inputs must be labeled
- **screen-readers**: Labels must be descriptive

## DEPENDENCIES

No dependencies.

## NOTES

- Labels are critical for accessibility
- Screen readers use labels to announce inputs
- Placeholder text is NOT a label
- Use visible labels when possible
- `aria-label` for icon-only buttons
