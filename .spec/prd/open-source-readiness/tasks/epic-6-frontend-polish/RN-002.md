## CRITICAL CONSTRAINTS

### MUST
- Refactor `useUrlValue` to NOT use local state + useEffect for URL syncing
- Return URL values directly from URL reading functions
- Use `useRouter` events for URL updates, not useEffect syncing

### NEVER
- Use `useState` + `useEffect` to sync URL values to local state

### STRICTLY
- URL values are source of truth, not derived state

### Anti-Pattern
```typescript
// ❌ WRONG: State syncing with useEffect
export function useUrlValue(param: string) {
  const [value, setValue] = useState<string>()

  useEffect(() => {
    const params = useLocalSearchParams()
    setValue(params[param] as string)
  }, [param])

  return value
}
```

### Correct Pattern
```typescript
// ✅ GOOD: Direct URL reading
import { useLocalSearchParams } from 'expo-router'

export function useUrlValue(param: string): string | undefined {
  const params = useLocalSearchParams()
  return params[param] as string | undefined
}

// ✅ BETTER: With router update handling
import { useLocalSearchParams, useNavigation } from 'expo-router'
import { useFocusEffect } from '@react-navigation/native'

export function useUrlValue(param: string): string | undefined {
  const params = useLocalSearchParams()
  const navigation = useNavigation()
  const [value, setValue] = useState<string | undefined>()

  useFocusEffect(
    useCallback(() => {
      setValue(params[param] as string | undefined)
    }, [params, param])
  )

  return value
}
```

## CODING STANDARDS

- **hooks**: URL is source of truth, not derived state
- **react**: Avoid useState + useEffect syncing patterns

## NOTES

- The anti-pattern causes stale state bugs
- URL parameters should be read directly when needed
- If caching is needed, use React Navigation's focus events
- Consider if you even need a hook for this — might be over-engineering
- Check where this hook is used and understand the usage pattern
- This is a common React Native anti-pattern to avoid
