# RN-011: Remove or Fix Dead `useLocalSearchParams` Hook

> Task ID: RN-011
> Type: REFACTOR
> Priority: P1
> Estimate: 10 minutes
> Assignee: react-native-implementer

## CRITICAL CONSTRAINTS

### MUST
- Verify if a custom `useLocalSearchParams` hook exists
- Check if it's a duplicate of expo-router's built-in hook
- If duplicate wrapper: delete and use expo-router's directly

### NEVER
- Wrap built-in hooks unnecessarily

### STRICTLY
- No duplicate wrapper hooks

## SPECIFICATION

**Objective:** Remove duplicate `useLocalSearchParams` hook if it's just wrapping expo-router.

**Success looks like:** Code uses expo-router's `useLocalSearchParams` directly.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | Custom `useLocalSearchParams` exists | Checking implementation | Either deleted or justified | Decision documented |
| 2 | If simple wrapper | Deleting | File removed | `! ls hooks/useLocalSearchParams.ts` |
| 3 | After removal | Updating imports | Uses expo-router directly | Imports from `expo-router` |

### Wrong Pattern (If This Exists)
```typescript
// ❌ WRONG: Unnecessary wrapper
// hooks/useLocalSearchParams.ts
export { useLocalSearchParams } from 'expo-router'

// or worse:
import { useLocalSearchParams as _useLocalSearchParams } from 'expo-router'
export const useLocalSearchParams = _useLocalSearchParams
```

### Correct Pattern
```typescript
// ✅ GOOD: Use expo-router directly
import { useLocalSearchParams } from 'expo-router'

export function MyScreen() {
  const params = useLocalSearchParams()
  // ...
}
```
