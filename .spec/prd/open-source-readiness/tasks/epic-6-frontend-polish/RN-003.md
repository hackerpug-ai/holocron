### Anti-Pattern
```typescript
// ❌ WRONG: State syncing with useEffect
export function useListValues() {
  const [items, setItems] = useState<string[]>([])

  useEffect(() => {
    const data = getSomeData()
    setItems(data)
  }, [])

  return items
}
```

### Correct Pattern
```typescript
// ✅ GOOD: Direct data return
import { useQuery } from 'convex/react'

export function useListValues() {
  const items = useQuery(api.myModule.listItems)

  return items ?? []
}

// ✅ ALTERNATIVE: If using non-Convex source
export function useListValues() {
  // Return derived value directly, no state
  return useMemo(() => {
    return computeListFromSource()
  }, [/* dependencies */])
}
```

## CODING STANDARDS

- **hooks**: Source data is truth, not derived state
- **react**: Avoid useState + useEffect syncing patterns

## DEPENDENCIES

No dependencies.

## NOTES

- This is the same anti-pattern as `useUrlValue`
- State syncing causes stale data bugs and race conditions
- If data comes from Convex, use `useQuery` directly
- If data is computed, use `useMemo`
- If data is from another async source, create a proper async hook
- Check where this hook is used and understand what data source it wraps
