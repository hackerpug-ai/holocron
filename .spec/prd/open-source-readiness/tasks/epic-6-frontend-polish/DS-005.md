### MUST
- Verify all interactive elements use proper state colors
- Check Pressable, TouchableOpacity, Button components

### NEVER
- Hardcode interactive state colors

### STRICTLY

### Verification Checklist

1. **Pressable Components**
   - [ ] Uses `semantic.color.primary.pressed` for pressed state
   - [ ] Uses `semantic.color.primary.hover` for hover state (web)
   - [ ] Uses `semantic.color.primary.disabled` for disabled state

2. **Button Components**
   - [ ] Uses Paper Button (handles states automatically)
   - [ ] If custom, uses proper state colors

3. **TouchableOpacity**
   - [ ] Uses theme tokens for opacity or color change

### Correct Pattern
```typescript
// ✅ GOOD: Using semantic color states
import { Pressable } from 'react-native'
import { useSemanticTheme } from '@/hooks/use-semantic-theme'

const { semantic } = useSemanticTheme()

<Pressable
  style={({ pressed }) => ({
    backgroundColor: pressed
      ? semantic.color.primary.pressed
      : semantic.color.primary.default,
  })}
>
  <Text>Press Me</Text>
</Pressable>

// ✅ GOOD: Using Paper Button (automatic states)
import { Button } from 'react-native-paper'

<Button mode="contained">Press Me</Button>

// ✅ GOOD: Full state handling
<Pressable
  style={({ pressed, hovered, focused }) => ({
    backgroundColor: disabled
      ? semantic.color.primary.disabled
      : pressed
      ? semantic.color.primary.pressed
      : hovered
      ? semantic.color.primary.hover
      : semantic.color.primary.default,
    borderColor: focused
      ? semantic.color.primary.focus
      : semantic.color.border.default,
  })}
>
```
