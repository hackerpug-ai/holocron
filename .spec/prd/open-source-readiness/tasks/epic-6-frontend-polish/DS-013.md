### Verification Checklist

1. **Native Elements**
   - [ ] Buttons are focusable
   - [ ] Inputs are focusable
   - [ ] Links are focusable

2. **Custom Components**
   - [ ] Have `tabIndex` if interactive
   - [ ] Handle keyboard events (Enter, Space)
   - [ ] Show visible focus indicator

3. **Focus Management**
   - [ ] Focus moves logically
   - [ ] Focus trap in modals
   - [ ] Focus returns after close

### Correct Pattern
```typescript
// ✅ GOOD: Native button (keyboard accessible by default)
<button onClick={handleClick}>Click me</button>

// ✅ GOOD: Custom interactive with keyboard
<div
  role="button"
  tabIndex={0}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault()
      handleClick()
    }
  }}
  onClick={handleClick}
>
  Custom Button
</div>

// ✅ GOOD: Focusable component
import { useFocusable } from '@react-native-web/community'

const FocusableButton = ({ onPress }) => {
  const ref = useRef(null)
  const { focusProps } = useFocusable({
    onFocus: () => console.log('Focused'),
  })

  return (
    <button
      ref={ref}
      {...focusProps}
      onClick={onPress}
    >
      Click me
    </button>
  )
}

// ❌ BAD: Div with onClick but no keyboard support
<div onClick={handleClick}>Click me</div>
```

### Finding Issues
```bash
# Find divs with onClick but no keyboard support
grep -rn "div.*onClick" components/web/ | grep -v "onKeyDown\|tabIndex"

# Check for tabIndex on interactive elements
grep -rn "tabIndex" components/web/
```

### Manual Testing
1. Tab through the page — can you reach all interactive elements?
2. Press Enter/Space on focused elements — do they activate?
3. Is the focus indicator visible?
4. Does focus move in a logical order?

## CODING STANDARDS

- **accessibility**: All interactions must work via keyboard
- **focus**: Visible focus indicators required
