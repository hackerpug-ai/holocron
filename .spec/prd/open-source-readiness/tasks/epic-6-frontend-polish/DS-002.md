### Finding Hardcoded Spacing
```bash
# Find padding values
grep -rn "padding:[ ]*[0-9]" app/ components/

# Find margin values
grep -rn "margin:[ ]*[0-9]" app/ components/

# Find paddingHorizontal/Y
grep -rn "padding" app/ components/ | grep -E "[0-9]+"
```

### Replacement Pattern
```typescript
// ❌ BEFORE: Hardcoded spacing
const styles = StyleSheet.create({
  container: {
    padding: 16,
    paddingHorizontal: 8,
    marginBottom: 24,
  },
})

// ✅ AFTER: Theme tokens
const { semantic } = useSemanticTheme()

const styles = useMemo(() => StyleSheet.create({
  container: {
    padding: semantic.space.lg,
    paddingHorizontal: semantic.space.sm,
    marginBottom: semantic.space.xl,
  },
}), [semantic])
```

### Spacing Scale Reference
| Token | Value | Use Case |
|-------|-------|----------|
| `xs` | 4px | Tiny gaps |
| `sm` | 8px | Small gaps |
| `md` | 12px | Medium gaps |
| `lg` | 16px | Standard padding |
| `xl` | 24px | Large padding |
| `2xl` | 32px | Extra large padding |
| `3xl` | 48px | Section spacing |

### Common Replacements
| Hardcoded | Theme Token |
|-----------|-------------|
| `4` | `semantic.space.xs` |
| `8` | `semantic.space.sm` |
| `12` | `semantic.space.md` |
| `16` | `semantic.space.lg` |
| `24` | `semantic.space.xl` |
| `32` | `semantic.space['2xl']` |
| `48` | `semantic.space['3xl']` |

## CODING STANDARDS

- **theme**: All spacing uses semantic tokens
- **consistency**: Use predefined scale, not arbitrary numbers

## DEPENDENCIES

No dependencies.

## NOTES

- Spacing is one of the most common hardcoded values
- Check both inline styles and StyleSheet.create
- Don't forget about gap, paddingHorizontal, etc.
- Some values might not match the scale exactly — use closest
- This refactor improves consistency across the app
