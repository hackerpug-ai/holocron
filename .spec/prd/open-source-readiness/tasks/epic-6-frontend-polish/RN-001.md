# RN-001: Remove `.bak` Files from Repository

> Task ID: RN-001
> Type: CLEANUP
> Priority: P0
> Estimate: 10 minutes
> Assignee: integrator

## CRITICAL CONSTRAINTS

### MUST
- Remove all `.bak` files from the repository
- Add `.bak` to `.gitignore` to prevent future commits
- Verify no `.bak` files remain in git history

### NEVER
- Commit `.bak` files after cleanup

### STRICTLY
- Repository must be free of backup artifacts

## SPECIFICATION

**Objective:** Remove all `.bak` files that were accidentally committed to the repository.

**Success looks like:** Zero `.bak` files in the working directory and `.gitignore` updated.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | `.bak` files in repo | Running find | All removed | `find . -name "*.bak" -type f` returns empty |
| 2 | `.gitignore` exists | Adding .bak pattern | Pattern added | `grep "\.bak" .gitignore` |
| 3 | After cleanup | Checking git status | No .bak files | `git ls-files "*.bak"` returns empty |

## TEST CRITERIA

| # | Boolean Statement | Maps To AC | Verify | Status |
|---|-------------------|------------|--------|--------|

| 2 | .bak in gitignore when checking | AC-69 | `grep -q "\.bak$" .gitignore` | [ ] TRUE [ ] FALSE |
| 3 | No .bak files tracked by git when checking | AC-69 | `! git ls-files "*.bak" | head -1` | [ ] TRUE [ ] FALSE |

## GUARDRAILS

### WRITE-ALLOWED
- `git rm` to remove tracked `.bak` files
- `.gitignore` (MODIFY) - Add `.bak` pattern

### WRITE-PROHIBITED
- Committing new `.bak` files

## DESIGN

### References
- PRD Section 2.10, AC-69

### Cleanup Steps
1. Find all `.bak` files:
   ```bash
   find . -name "*.bak" -type f
   ```

2. Remove them from git:
   ```bash
   git rm *.bak
   # or recursively:
   find . -name "*.bak" -type f -exec git rm {} \;
   ```

3. Add to `.gitignore`:
   ```
   # Backup files
   *.bak
   ```

4. Commit the cleanup:
   ```bash
   git add .gitignore
   git commit -m "chore: remove .bak files and add to gitignore"
   ```

## CODING STANDARDS

- **cleanup**: Remove artifacts, don't hide them
- **git**: Clean history, don't accumulate junk

## DEPENDENCIES

No dependencies.

## NOTES

- `.bak` files are editor backup files that shouldn't be in version control
- This is a quick cleanup task but important for repo hygiene
- Check both root and subdirectories
- Common locations: `convex/`, `app/`, `components/`
- Consider adding other editor backup patterns: `*~`, `*.swp`, `*.swo`
- This task is P0 because committed artifacts look unprofessional
