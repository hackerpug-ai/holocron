### Verification Checklist

1. **Structural Elements**
   - [ ] Uses `<header>` for headers
   - [ ] Uses `<nav>` for navigation
   - [ ] Uses `<main>` for main content
   - [ ] Uses `<article>` for articles
   - [ ] Uses `<section>` for sections
   - [ ] Uses `<footer>` for footers

2. **Interactive Elements**
   - [ ] Uses `<button>` for buttons
   - [ ] Uses `<input>` for inputs
   - [ ] Uses `<form>` for forms
   - [ ] No `<div onClick={...}>` patterns

3. **Text Elements**
   - [ ] Uses `<h1>`-`<h6>` for headings
   - [ ] Uses `<p>` for paragraphs
   - [ ] Uses `<label>` for labels

### Correct Pattern
```typescript
// ❌ BAD: Div soup
<div className="header">
  <div className="title">Title</div>
  <div className="button" onClick={handleClick}>Click</div>
</div>

// ✅ GOOD: Semantic HTML
<header className="container">
  <h1>Title</h1>
  <button onClick={handleClick}>Click</button>
</header>

// ❌ BAD: Div button
<div role="button" onClick={handleClick}>Press</div>

// ✅ GOOD: Button element
<button onClick={handleClick}>Press</button>

// ✅ GOOD: Semantic structure
<article>
  <header>
    <h2>Article Title</h2>
  </header>
  <section>
    <p>Article content...</p>
  </section>
  <footer>
    <small>Footer info</small>
  </footer>
</article>
```

### Finding Issues
```bash
# Find div-based buttons
grep -rn 'div.*role="button"' components/web/
grep -rn 'div.*onClick' components/web/

# Check for semantic elements
grep -rn '<header>\|<nav>\|<main>' components/web/
```
