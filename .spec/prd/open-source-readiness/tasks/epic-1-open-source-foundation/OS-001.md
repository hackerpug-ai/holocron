## CRITICAL CONSTRAINTS

### MUST
- Rotate ALL API keys that were previously committed to .env or .env.local
- Use git-filter-repo to purge keys from entire git history
- Verify no keys remain in history after purge
- Commit new .env.example with safe placeholder names

### NEVER
- Commit actual API keys during this task
- Use git filter-branch (deprecated, use git-filter-repo instead)
- Push to remote before verifying the purge locally

### STRICTLY
- This is a destructive operation - create a backup branch before starting
- All team members must be notified before force-pushing rewritten history

## SPECIFICATION

**Objective:** Remove all API keys from git history and rotate compromised credentials

**Success looks like:** `git log --all -p -S "sk-"` returns zero results, .env.example uses safe placeholder names, and all previously committed keys have been rotated at their respective providers.

### Code Pattern
```bash
# Backup before destructive operation
git branch backup-before-purge

# Install git-filter-repo
pip install git-filter-repo

# Purge .env files from history
git filter-repo --path .env --path .env.local --invert-paths

# Verify purge worked
git log -p -S "sk-" --all

# If clean, force push (NOTIFY TEAM FIRST)
# git push origin --force --all
```

### Anti-pattern (DO NOT)
```bash
# WRONG: Using deprecated git filter-branch
git filter-branch --tree-filter 'rm -f .env' HEAD

# WRONG: Not creating backup before destructive operations
git filter-repo --path .env --invert-paths  # No backup!
```
