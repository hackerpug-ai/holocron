MUST:
- MUST short-circuit specialist dispatch when directResponse is set AND (intent='conversation' OR queryShape='ambiguous') AND confidence != 'low'
- MUST call setPendingIntent when queryShape='ambiguous' and clarificationDepth < 1
- MUST call clearPendingIntent on non-ambiguous turn completion
- MUST enforce clarificationDepth hard cap deterministically (not via prompt alone)
- MUST prepend 'The user is mid-request. Their original intent was {pendingIntent}' to triage input when pending state is active and fresh

SPECIFICATION
--------------------------------------------------------------------------------

**Objective**: Wire pending-state persistence and the ambiguous short-circuit into `agent.ts` so that (1) ambiguous triage emits a `directResponse` and sets pending state, (2) the next turn rehydrates pending intent into the triage preamble, (3) non-ambiguous turns clear pending state, (4) `clarificationDepth` is enforced to 1.

**Success looks like**: Manual smoke: send "find me a coach" → receive clarifying question → respond "SF" → agent completes original recommendation. Unit tests cover all 4 paths.

--------------------------------------------------------------------------------
BACKGROUND
--------------------------------------------------------------------------------

**Problem**: INT-006 wired the new triage into agent.ts, but it doesn't yet handle the ambiguous case. Without this, even if triage emits `queryShape='ambiguous'` with a `directResponse`, the orchestrator ignores it and dispatches to a specialist anyway. Also, pending state is never read or written.

**Why it matters**: This is the keystone for multi-turn coherence. Without it, clarification rounds lose context and the user has to repeat themselves.

**Current state**: `agent.ts` post-INT-006 calls triage + records telemetry but always dispatches to a specialist.

**Desired state**: `agent.ts` checks the triage result: if ambiguous + directResponse + not at cap → persist directResponse as assistant message, set pending state, return. Next turn reads pending state into preamble.

CODE PATTERN (Reference)

**Source**: `convex/chat/agent.ts` existing triage dispatch

```ts
import { isPendingExpired } from '../conversations/mutations';
import { internal } from '../_generated/api';

// Read pending state from conversation doc
const conv = await ctx.runQuery(internal.conversations.queries.get, { conversationId });
const isPendingFresh = conv.pendingSince && !isPendingExpired(conv.pendingSince);
const preamble = isPendingFresh
  ? `The user is mid-request. Their original intent was ${conv.pendingIntent} with shape ${conv.pendingQueryShape}.`
  : undefined;

// Pass preamble to classifyIntent
const triage = await classifyIntent(messages, regexHint, preamble);

// Compute clarificationDepth from recent telemetry
const clarificationDepth = await computeClarificationDepth(ctx, conversationId);

// Ambiguous short-circuit
if (triage.queryShape === 'ambiguous' && triage.directResponse && clarificationDepth < 1 && triage.confidence !== 'low') {
  await ctx.runMutation(internal.conversations.mutations.setPendingIntent, {
    conversationId,
    intent: triage.intent,
    queryShape: triage.queryShape,
  });
  await persistAssistantMessage(ctx, conversationId, {
    role: 'assistant',
    content: triage.directResponse,
    card_data: { card_type: 'clarification', question: triage.directResponse, answered: false },
  });
  return; // no specialist dispatch
}

// Clamp over-depth ambiguous → best-guess factual
const effectiveShape = (triage.queryShape === 'ambiguous' && clarificationDepth >= 1) ? 'factual' : triage.queryShape;

// Non-ambiguous path clears pending state
if (effectiveShape !== 'ambiguous' && conv.pendingIntent) {
  await ctx.runMutation(internal.conversations.mutations.clearPendingIntent, { conversationId });
}

// ...continue to specialist dispatch with effectiveShape...
```

**Anti-pattern**: Do NOT call `ctx.db.patch` from an action; do NOT trust prompt-level caps alone for `clarificationDepth`.

- Pending state rehydrate is a READ from the conversation doc before calling triage
