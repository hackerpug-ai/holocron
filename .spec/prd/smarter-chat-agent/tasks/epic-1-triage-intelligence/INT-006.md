SPECIFICATION
--------------------------------------------------------------------------------

**Objective**: Wire the regex pre-filter, the refactored triage, and the telemetry mutation together inside `agent.ts` so that every turn emits an `agentTelemetry` row with the correct `classificationSource` and the regex wins on recommendation-pattern disagreements.

**Success looks like**: Manual smoke via dev chat shows the canonical failing query producing `queryShape=recommendation` and an `agentTelemetry` row with `classificationSource in {regex, llm}`; existing chat flows unaffected.

--------------------------------------------------------------------------------
BACKGROUND
--------------------------------------------------------------------------------

**Problem**: Pieces exist (regex, new triage parser, telemetry writer) but they're not yet composed in the agent orchestrator. Without this task, nothing observes or acts on the new capability.

**Why it matters**: This is the epic's keystone — it takes a schema, a pure module, a prompt, and a mutation and turns them into actual behavior the user can observe via telemetry.

**Current state**: `agent.ts` calls `classifyIntent` directly; no regex pre-filter; no telemetry recording.

**Desired state**: `agent.ts` calls `regexClassify()` first, passes the hint into `classifyIntent`, reconciles the two results deterministically, records a telemetry row via `ctx.runMutation`, and dispatches to specialists with the final `queryShape` in scope.

CODE PATTERN (Reference)
--------------------------------------------------------------------------------

**Source**: `convex/chat/agent.ts` existing classifyIntent call site

```ts
import { regexClassify } from './triageRegex';
import { internal } from '../_generated/api';

// inside run() action, before specialist dispatch
const userContent = extractUserContent(messages);
const regexHint = regexClassify(userContent);
const start = Date.now();
const llmResult = await classifyIntent(messages, regexHint, pendingIntent);
const llmDurationMs = Date.now() - start;

// Deterministic reconciliation: regex wins on recommendation disagreement
const regexSaysRec = regexHint?.queryShape === 'recommendation';
const llmSaysDifferent = llmResult.queryShape !== 'recommendation';
const finalShape = regexSaysRec && llmSaysDifferent ? 'recommendation' : llmResult.queryShape;
const classificationSource = regexSaysRec && llmSaysDifferent ? 'regex' : 'llm';

// Record telemetry via runMutation (action -> mutation)
await ctx.runMutation(internal.chat.telemetryMutations.recordTriage, {
  conversationId,
  messageId,
  intent: llmResult.intent,
  queryShape: finalShape,
  confidence: llmResult.confidence,
  reasoning: llmResult.reasoning,
  classificationSource,
  regexMatchPattern: regexHint?.matchedPattern,
  rawLlmResponse: llmResult.rawLlmResponse,
  llmDurationMs,
  totalDurationMs: Date.now() - turnStart,
});
```

**Anti-pattern**: Do NOT call `ctx.db.insert` from an action; do NOT skip telemetry recording on fallback paths; do NOT override queryShape for non-recommendation disagreements.

**Interaction notes**:
- Regex runs first, result feeds LLM as hint AND is used to override on disagreement
- `recordTriage` is called via `ctx.runMutation` because `agent.ts` is an action
