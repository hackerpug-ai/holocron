BACKGROUND
--------------------------------------------------------------------------------

**Problem**: The LLM triage alone is demonstrably unreliable on recommendation patterns — it picks `deep_research` on phrases like "highly rated" because they *sound* comprehensive. A deterministic backstop is needed to catch the highest-frequency failure pattern.

**Why it matters**: This is the single highest-leverage intervention in the entire PRD. Fixing the canonical failing case depends on the regex overriding the LLM when they disagree.

**Current state**: No pre-filter exists; `convex/chat/triage.ts` calls the LLM directly for every turn.

**Desired state**: A pure, fast, Convex-import-free module that catches ~80% of recommendation patterns with near-zero false positives.

```ts
export type QueryShape = 'factual' | 'recommendation' | 'comprehensive' | 'exploratory' | 'ambiguous';

interface RegexResult {
  queryShape: QueryShape;
  confidence: 'high' | 'medium';
  matchedPattern: string;
}

const REC_PATTERNS: { regex: RegExp; label: string }[] = [
  { regex: /\b(find me|get me|show me)\s+\d+\b/i, label: 'find_me_n' },
  { regex: /\b(top|best)\s+\d+\b/i, label: 'top_n' },
  { regex: /highly\s+rated/i, label: 'highly_rated' },
  { regex: /\breferrals?\s+for\b/i, label: 'referrals_for' },
  { regex: /\bwho\s+should\s+I\s+hire\b/i, label: 'who_should_hire' },
  { regex: /\bwhere\s+can\s+I\s+find\b/i, label: 'where_can_find' },
  { regex: /\bprovide\s+\d+(-\d+)?\b/i, label: 'provide_n' },
];

export function regexClassify(content: string): RegexResult | undefined {
  for (const p of REC_PATTERNS) {
    if (p.regex.test(content)) {
      return { queryShape: 'recommendation', confidence: 'high', matchedPattern: p.label };
    }
  }
  return undefined;
}
```

**Anti-pattern**: Do NOT import `ctx` or `v` from convex; do NOT call the LLM from here; do NOT emit intents outside the known union; do NOT try to classify non-recommendation patterns (keep it high-precision, narrow).

**Interaction notes**:
- Regex is a HIGH-PRECISION backstop, not a full classifier — return undefined when no pattern fires
- When both regex and LLM fire with conflicting queryShape, regex wins (logged to telemetry as `classificationSource='regex'`)
