--------------------------------------------------------------------------------
BACKGROUND
--------------------------------------------------------------------------------

**Problem**: The triage classifier currently returns only `{ intent, confidence, reasoning }`. There is no `queryShape` field, so routing cannot distinguish "find me 5 coaches" from "explain coaches thoroughly" even when the intent is identical.

**Why it matters**: `queryShape` drives tool selection in Decision Table A. Without it, Epic 2's `find_recommendations` tool has no hook to differentiate recommendation queries from comprehensive research queries.

**Current state**: `convex/chat/triage.ts` emits an `IntentCategory`-only result with a JSON-parsing regex and a `fallbackResult()` helper.

**Desired state**: Triage returns 5 fields, validates `queryShape` against a runtime allow-list, and falls back to `factual` on any invalid/missing value without throwing.

CODE PATTERN (Reference)

**Source**: `convex/chat/triage.ts` existing IntentCategory validation

```ts
export type QueryShape = 'factual' | 'recommendation' | 'comprehensive' | 'exploratory' | 'ambiguous';
export const VALID_QUERY_SHAPES: readonly QueryShape[] = [
  'factual', 'recommendation', 'comprehensive', 'exploratory', 'ambiguous',
] as const;

function coerceQueryShape(raw: unknown): QueryShape {
  return (typeof raw === 'string' && (VALID_QUERY_SHAPES as readonly string[]).includes(raw))
    ? raw as QueryShape
    : 'factual';
}

// inside classifyIntent handler, after JSON.parse:
const parsed = safeParseJson(raw);
if (!parsed) return fallbackResult();
return {
  intent: coerceIntent(parsed.intent),
  queryShape: coerceQueryShape(parsed.queryShape),
  confidence: parsed.confidence ?? 'low',
  reasoning: parsed.reasoning,
  directResponse: parsed.directResponse,
  rawLlmResponse: raw.slice(0, 2000),
};
```

**Anti-pattern**: Do NOT throw on invalid shape; do NOT skip the `VALID_QUERY_SHAPES` allow-list check; do NOT forget to truncate rawLlmResponse.
