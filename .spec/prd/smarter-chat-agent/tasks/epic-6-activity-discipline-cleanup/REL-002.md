CODE PATTERN (Reference)

**Source**: `convex/crons.ts` existing daily/interval entries

```ts
// convex/crons.ts
import { cronJobs } from 'convex/server'
import { internal } from './_generated/api'

const crons = cronJobs()

crons.daily(
  'cleanup agentTelemetry',
  { hourUTC: 7, minuteUTC: 0 },
  internal.chat.telemetryMutations.deleteOldTelemetry,
  { olderThanMs: 90 * 24 * 60 * 60 * 1000 },
)

export default crons

// convex/chat/telemetryMutations.ts — deleteOldTelemetry implementation
export const deleteOldTelemetry = internalMutation({
  args: { olderThanMs: v.number() },
  returns: v.object({ deleted: v.number() }),
  handler: async (ctx, { olderThanMs }) => {
    const cutoff = Date.now() - olderThanMs
    let deleted = 0
    const BATCH = 1000
    while (deleted < BATCH) {
      const batch = await ctx.db
        .query('agentTelemetry')
        .withIndex('by_createdAt', q => q.lt('createdAt', cutoff))
        .take(BATCH)
      if (batch.length === 0) break
      for (const row of batch) await ctx.db.delete(row._id)
      deleted += batch.length
      if (batch.length < BATCH) break
    }
    return { deleted }
  },
})
```

**Anti-pattern**: Deleting in a single giant transaction; dropping the whole table.

**Interaction notes**:
- Convex has transaction size limits — batch deletes to 1000 rows at a time
- Cron calls an internalMutation that loops until done or batch cap
