## Draft Description

Create feedback mutations in convex/feedback/mutations.ts.
Use existing `userFeedback` and `userFeedbackAt` fields on feedItems table.
Create `record` mutation: takes feedItemId, sentiment ('up'|'down'), updates feedItems userFeedback/userFeedbackAt.
Create `undo` mutation: takes feedItemId, clears userFeedback/userFeedbackAt fields.
Use getAuthUserId from auth utilities for user authentication.
Create convex/feedback/index.ts to export mutations.
Use v.string() for feedItemId, v.union(v.literal('up'), v.literal('down')) for sentiment.
Handle case where feedItemId not found (graceful no-op).
Update timestamp to Date.now() on record.
Export { record, undo } from index.ts.
